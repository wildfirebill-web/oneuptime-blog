# How to Automate IPv6 Address Planning with Ansible

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Ansible, IPv6, Address Planning, IPAM, Network Automation, Jinja2

Description: A guide to using Ansible to automate IPv6 address allocation and planning, including CIDR calculation, documentation generation, and IPAM integration.

IPv6 address planning involves allocating /48s to sites, /64s to subnets, and tracking assignments. Ansible can automate the calculation, assignment, and documentation of IPv6 addresses across your infrastructure.

## Step 1: Define the Address Hierarchy

```yaml
# group_vars/all.yml - IPv6 address hierarchy definition
---
# Organization's allocated /32 prefix
ipv6_org_prefix: "2001:db8::/32"

# Site allocations: each site gets a /48 from the /32
# 2001:db8:0001::/48 = Site A
# 2001:db8:0002::/48 = Site B
# etc.
ipv6_sites:
  site_a:
    prefix: "2001:db8:0001::/48"
    location: "US East"
  site_b:
    prefix: "2001:db8:0002::/48"
    location: "EU West"

# Subnet allocations within each site (using /64 from the /48)
ipv6_subnet_map:
  site_a:
    management: "2001:db8:0001:0001::/64"
    servers:    "2001:db8:0001:0002::/64"
    dmz:        "2001:db8:0001:0003::/64"
    clients:    "2001:db8:0001:0010::/64"
  site_b:
    management: "2001:db8:0002:0001::/64"
    servers:    "2001:db8:0002:0002::/64"
```

## Step 2: Calculate and Assign Host Addresses

```yaml
# assign-ipv6-addresses.yml - Calculate and assign IPv6 addresses from subnets
---
- name: Calculate and assign IPv6 addresses
  hosts: localhost
  gather_facts: false

  vars:
    site: "site_a"
    server_subnet: "{{ ipv6_subnet_map[site].servers }}"

    # Host assignments within the servers /64
    server_assignments:
      - host: "web-01"
        offset: 10  # 2001:db8:0001:0002::10
      - host: "web-02"
        offset: 11
      - host: "db-01"
        offset: 20
      - host: "db-02"
        offset: 21

  tasks:
    - name: Calculate IPv6 address for each server
      ansible.builtin.set_fact:
        server_ipv6_map: >-
          {{
            server_ipv6_map | default({}) | combine({
              item.host: (server_subnet | ansible.netcommon.ipaddr(item.offset) | ansible.netcommon.ipaddr('address'))
            })
          }}
      loop: "{{ server_assignments }}"

    - name: Display the address allocation
      ansible.builtin.debug:
        var: server_ipv6_map
```

## Step 3: Update Inventory with Calculated Addresses

```yaml
# update-inventory.yml - Write IPv6 assignments back to inventory variables
---
  tasks:
    - name: Write host IPv6 assignments to group_vars
      ansible.builtin.template:
        src: ipv6-assignments.yml.j2
        dest: "group_vars/servers/ipv6-assignments.yml"
        mode: "0644"
```

Template:
```yaml
# templates/ipv6-assignments.yml.j2
# Auto-generated by Ansible - do not edit manually
# Generated: {{ ansible_date_time.iso8601 }}
host_ipv6_addresses:
{% for host, ipv6 in server_ipv6_map.items() %}
  {{ host }}: "{{ ipv6 }}"
{% endfor %}
```

## Step 4: Generate IPv6 Address Documentation

```yaml
# generate-ipv6-docs.yml - Create a CSV report of all IPv6 assignments
---
  tasks:
    - name: Generate IPv6 address plan CSV
      ansible.builtin.template:
        src: ipv6-plan.csv.j2
        dest: "/tmp/ipv6-address-plan.csv"

    - name: Display the generated plan
      ansible.builtin.command:
        cmd: cat /tmp/ipv6-address-plan.csv
      register: csv_output
      changed_when: false

    - name: Print the address plan
      ansible.builtin.debug:
        msg: "{{ csv_output.stdout_lines }}"
```

## Step 5: Integrate with NetBox IPAM

```yaml
# netbox-ipv6.yml - Create IPv6 prefixes and IPs in NetBox
---
  tasks:
    - name: Create IPv6 prefix in NetBox
      netbox.netbox.netbox_prefix:
        netbox_url: "https://netbox.example.com"
        netbox_token: "{{ netbox_token }}"
        data:
          prefix: "{{ ipv6_subnet_map.site_a.servers }}"
          site: "site-a"
          description: "Site A Servers IPv6 Subnet"
          status: active
        state: present

    - name: Assign IPv6 address to a host in NetBox
      netbox.netbox.netbox_ip_address:
        netbox_url: "https://netbox.example.com"
        netbox_token: "{{ netbox_token }}"
        data:
          address: "{{ server_ipv6_map['web-01'] }}/64"
          status: active
          dns_name: "web-01.example.com"
        state: present
```

## Run the Address Planning Playbook

```bash
# Generate address assignments
ansible-playbook assign-ipv6-addresses.yml

# Generate documentation
ansible-playbook generate-ipv6-docs.yml

# Sync to NetBox
ansible-playbook netbox-ipv6.yml
```

Automating IPv6 address planning with Ansible eliminates spreadsheet-based tracking, ensures consistency between your address plan and actual host configuration, and integrates naturally with IPAM tools like NetBox.
