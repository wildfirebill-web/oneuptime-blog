## What Is Vector Search

Vector search is a technique for finding semantically similar items by comparing numerical vector representations (embeddings) of data. Unlike traditional keyword search, which looks for exact or fuzzy string matches, vector search finds items based on meaning and context.

For example, searching for "automobile" can return documents containing "car" or "vehicle" because all three concepts have similar vector representations in a well-trained embedding model.

MongoDB Atlas Vector Search brings this capability directly into MongoDB, allowing you to store embeddings alongside your data and perform vector similarity searches using the same aggregation pipeline you use for other queries.

## How Vector Embeddings Work

Embeddings are fixed-length arrays of floating-point numbers generated by machine learning models. A text passage like "MongoDB is a document database" might be represented as a 1536-dimensional vector such as `[0.023, -0.014, 0.089, ...]`.

When you embed a query, you get a similar vector. The search then finds documents whose stored vectors are closest to the query vector using a distance metric such as cosine similarity or Euclidean distance.

Common embedding sources include:
- OpenAI `text-embedding-ada-002` or `text-embedding-3-small`
- Hugging Face sentence transformers
- Google Vertex AI embedding models
- Cohere Embed

## Setting Up Atlas Vector Search

### Step 1 - Store Vector Embeddings in Documents

First, generate embeddings for your data and store them in a field in your documents:

```javascript
db.articles.insertMany([
  {
    title: "Introduction to MongoDB",
    content: "MongoDB is a NoSQL document database...",
    embedding: [0.023, -0.014, 0.089, /* ... 1536 dimensions */]
  },
  {
    title: "Getting Started with Atlas",
    content: "Atlas is MongoDB's cloud database platform...",
    embedding: [0.031, -0.009, 0.076, /* ... 1536 dimensions */]
  }
])
```

### Step 2 - Create a Vector Search Index

Vector Search indexes are created in the Atlas UI or via the Atlas CLI. They are separate from regular MongoDB indexes. The index definition specifies which field contains the vectors and the number of dimensions:

```json
{
  "fields": [
    {
      "type": "vector",
      "path": "embedding",
      "numDimensions": 1536,
      "similarity": "cosine"
    }
  ]
}
```

The supported similarity metrics are:
- `cosine` - angle between vectors, best for text embeddings
- `euclidean` - straight-line distance between vectors
- `dotProduct` - dot product of vectors (vectors must be normalized)

### Step 3 - Run a Vector Search Query

Use the `$vectorSearch` aggregation stage to perform a similarity search:

```javascript
const queryEmbedding = [0.025, -0.011, 0.091, /* ... 1536 dimensions */]

db.articles.aggregate([
  {
    $vectorSearch: {
      index: "vector_index",
      path: "embedding",
      queryVector: queryEmbedding,
      numCandidates: 100,
      limit: 5
    }
  },
  {
    $project: {
      title: 1,
      content: 1,
      score: { $meta: "vectorSearchScore" }
    }
  }
])
```

The `numCandidates` parameter controls the size of the candidate set during ANN (Approximate Nearest Neighbor) search. A higher value increases accuracy but also increases latency. The `limit` parameter controls how many results are returned.

## Pre-Filtering with Vector Search

You can combine vector search with metadata filters to narrow the search space before similarity is computed. This is useful for multi-tenant applications or category-specific searches:

```javascript
db.articles.aggregate([
  {
    $vectorSearch: {
      index: "vector_index",
      path: "embedding",
      queryVector: queryEmbedding,
      numCandidates: 100,
      limit: 5,
      filter: { category: "technology" }
    }
  },
  {
    $project: {
      title: 1,
      category: 1,
      score: { $meta: "vectorSearchScore" }
    }
  }
])
```

For filtering to work, the filter fields must also be defined in the vector search index definition:

```json
{
  "fields": [
    {
      "type": "vector",
      "path": "embedding",
      "numDimensions": 1536,
      "similarity": "cosine"
    },
    {
      "type": "filter",
      "path": "category"
    }
  ]
}
```

## Building a RAG Application with Atlas Vector Search

Retrieval-Augmented Generation (RAG) is a common pattern that uses vector search to find relevant context documents and passes them to an LLM. Here is a Python example using the PyMongo driver and OpenAI:

```python
import openai
from pymongo import MongoClient

client = MongoClient("mongodb+srv://...")
db = client["knowledge_base"]
collection = db["articles"]

def get_embedding(text):
    response = openai.embeddings.create(
        input=text,
        model="text-embedding-3-small"
    )
    return response.data[0].embedding

def search_articles(query, limit=5):
    query_vector = get_embedding(query)

    results = collection.aggregate([
        {
            "$vectorSearch": {
                "index": "vector_index",
                "path": "embedding",
                "queryVector": query_vector,
                "numCandidates": 100,
                "limit": limit
            }
        },
        {
            "$project": {
                "title": 1,
                "content": 1,
                "score": {"$meta": "vectorSearchScore"}
            }
        }
    ])

    return list(results)

# Search for semantically similar articles
results = search_articles("how to scale a database horizontally")
for r in results:
    print(r["title"], "-", r["score"])
```

## Atlas Vector Search vs. Text Indexes

| Feature | Atlas Vector Search | Text Indexes |
|---|---|---|
| Search type | Semantic similarity | Keyword matching |
| Supports typos | Via embedding model | Limited fuzzy match |
| Understands synonyms | Yes | No |
| Requires embeddings | Yes | No |
| Multi-language | Via multilingual models | Per-language text index |
| Score type | Cosine/Euclidean | TF-IDF relevance |

## Summary

MongoDB Atlas Vector Search enables semantic similarity search over vector embeddings stored in MongoDB collections. You create a vector search index on an embedding field, then use the `$vectorSearch` aggregation stage with a query vector to find the most similar documents. It supports pre-filtering on metadata fields and integrates naturally with AI frameworks like LangChain and LlamaIndex for building RAG pipelines and recommendation systems.
