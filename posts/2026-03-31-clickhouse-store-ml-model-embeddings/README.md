# How to Store ML Model Embeddings in ClickHouse

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: ClickHouse, Machine Learning, Embedding, Vector, Array, Data Storage

Description: Learn how to store and manage ML model embeddings in ClickHouse using Array(Float32) columns with best practices for schema design and ingestion.

---

Machine learning applications increasingly rely on embedding vectors - dense numerical representations of text, images, or other objects produced by models like BERT, OpenAI, or custom neural nets. Storing embeddings alongside structured metadata in ClickHouse lets you run fast similarity queries while retaining full SQL capabilities.

## Choosing the Right Data Type

Store embeddings as `Array(Float32)`. Using `Float32` instead of `Float64` halves storage size with negligible precision loss for similarity tasks.

```sql
CREATE TABLE text_embeddings
(
    id          UInt64,
    source      LowCardinality(String),
    text        String,
    model       LowCardinality(String),
    embedding   Array(Float32),
    dim         UInt16 MATERIALIZED length(embedding),
    created_at  DateTime DEFAULT now()
)
ENGINE = MergeTree()
PARTITION BY toYYYYMM(created_at)
ORDER BY (source, id);
```

The `dim` materialized column tracks embedding dimensionality without extra storage overhead, helping catch dimension mismatches.

## Inserting Embeddings

Embeddings are typically generated outside ClickHouse (Python, a model API) and then inserted via the HTTP or native interface.

```python
import clickhouse_connect

client = clickhouse_connect.get_client(host='localhost')

rows = [
    (1, 'articles', 'ClickHouse is fast', 'text-embedding-3-small', [0.12, 0.83, 0.45, 0.67]),
    (2, 'articles', 'Vector search at scale', 'text-embedding-3-small', [0.14, 0.80, 0.48, 0.70]),
]

client.insert('text_embeddings',
              rows,
              column_names=['id', 'source', 'text', 'model', 'embedding'])
```

Or using SQL with inline arrays:

```sql
INSERT INTO text_embeddings (id, source, text, model, embedding) VALUES
(3, 'articles', 'Fast analytical queries', 'text-embedding-3-small', [0.09, 0.77, 0.52, 0.63]);
```

## Validating Dimension Consistency

Guard against inserting wrong-dimension embeddings:

```sql
-- Check for dimension anomalies
SELECT model, length(embedding) AS dim, count() AS cnt
FROM text_embeddings
GROUP BY model, dim
ORDER BY model, cnt DESC;
```

You can also enforce dimensions via a constraint:

```sql
ALTER TABLE text_embeddings
    ADD CONSTRAINT check_dim CHECK length(embedding) = 1536;
```

## Querying Embeddings

Retrieve and compute similarity inline:

```sql
SELECT
    id,
    text,
    cosineDistance(embedding, [0.11, 0.82, 0.46, 0.68]) AS dist
FROM text_embeddings
WHERE source = 'articles'
ORDER BY dist ASC
LIMIT 5;
```

## Storage Optimization

For high-dimensional embeddings (768-4096 dims), storage becomes significant. Use `Delta` + `ZSTD` codec:

```sql
CREATE TABLE text_embeddings_compressed
(
    id        UInt64,
    embedding Array(Float32) CODEC(Delta, ZSTD(3)),
    created_at DateTime DEFAULT now()
)
ENGINE = MergeTree()
ORDER BY id;
```

This can reduce embedding storage by 30-50% depending on value distribution.

## Keeping Track of Model Versions

As models change, you need to track which embeddings were generated by which model version - cosine similarity is only meaningful within the same embedding space.

```sql
SELECT model, count() AS total, min(created_at) AS first_seen, max(created_at) AS last_seen
FROM text_embeddings
GROUP BY model
ORDER BY total DESC;
```

Use a separate table per model or a `model` partition key when re-embedding data at scale.

## Summary

ClickHouse stores ML embeddings efficiently in `Array(Float32)` columns. Use materialized columns to track dimensions, apply `Delta + ZSTD` codecs for storage efficiency, and include a `model` column to handle multiple embedding spaces. This foundation enables fast similarity search and hybrid metadata-plus-vector queries.
