# How to Configure OSPF Totally NSSA Areas

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: OSPF, NSSA, Routing, Networking, FRR, Cisco, IPv4, Configuration

Description: Learn how to configure OSPF Totally NSSA areas to reduce LSA flooding while allowing external route redistribution from non-OSPF routing domains.

---

A Totally NSSA (Not-So-Stubby Area) blocks Type 3, 4, and 5 LSAs from flooding into the area while still allowing NSSA-originated external routes (Type 7 LSAs). ABRs inject a default route instead, simplifying routing in spoke areas.

## OSPF Area Types Comparison

| Area Type | Blocks | Allows |
|-----------|--------|--------|
| Standard | Nothing | All LSAs |
| Stub | Type 5 (External) | Types 1, 2, 3, 4 |
| Totally Stub | Types 3, 4, 5 | Types 1, 2 + default |
| NSSA | Type 5 | Types 1, 2, 3, 4, 7 |
| Totally NSSA | Types 3, 4, 5 | Types 1, 2, 7 + default |

A Totally NSSA is the most restrictive while still allowing local external redistribution.

## FRR Configuration

### Area Border Router (ABR)

```
# On the ABR (connected to both the backbone and area 1)
router ospf
  ospf router-id 10.0.0.1
  area 1 nssa no-summary    ! "no-summary" makes it Totally NSSA
```

The `no-summary` flag is what makes an NSSA "totally" NSSA — it blocks Type 3 summary LSAs.

### ASBR Inside the NSSA (Redistributing External Routes)

```
# On the ASBR inside area 1 (e.g., redistributing a connected route or BGP)
router ospf
  ospf router-id 10.0.1.1
  area 1 nssa             ! Note: no "no-summary" on non-ABR routers

  redistribute connected  ! This generates Type 7 LSAs (NSSA external)
  redistribute static
```

### Example FRR Full Config

```
# ABR connecting backbone (area 0) to NSSA area 1
frr defaults traditional

router ospf
  ospf router-id 10.0.0.1

  ! Backbone interface
  interface eth0
    ip ospf area 0
    ip ospf hello-interval 10
    ip ospf dead-interval 40

  ! Area 1 interface
  interface eth1
    ip ospf area 1

  ! Configure area 1 as Totally NSSA
  area 1 nssa no-summary
  ! ABR injects a default route into area 1 automatically
```

## Cisco IOS Equivalent

```
router ospf 1
  router-id 10.0.0.1
  area 1 nssa no-summary    ! ABR: Totally NSSA
  ! or on the ASBR inside the area:
  ! area 1 nssa
```

## Verifying NSSA Configuration

```bash
# FRR
vtysh -c "show ip ospf"
# Look for: Area 1 is NSSA

vtysh -c "show ip ospf database nssa-external"
# Shows Type 7 LSAs generated from redistribution

vtysh -c "show ip route"
# The Totally NSSA area should see a default route from the ABR: 0.0.0.0/0 via O*IA
```

## Key Takeaways

- `area X nssa` enables NSSA; `area X nssa no-summary` makes it Totally NSSA (blocks Type 3 LSAs).
- Add `no-summary` only on the ABR, not on ASBR or internal routers within the area.
- Totally NSSA routers in the area receive a single default route from the ABR instead of summary LSAs.
- Type 7 LSAs (generated by redistribution in the NSSA) are converted to Type 5 at the ABR for flooding to the backbone.
