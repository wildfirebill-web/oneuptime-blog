# How to Configure DNS Servers in /etc/resolv.conf on Linux

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Linux, DNS, IPv4, Network Configuration, resolv.conf, Name Resolution

Description: Configure DNS nameservers in /etc/resolv.conf on Linux, prevent DHCP or NetworkManager from overwriting it, and use systemd-resolved for modern systems.

## Introduction

`/etc/resolv.conf` tells the C library resolver which DNS servers to query and what domain search suffixes to use. On modern systems it is often managed by NetworkManager or systemd-resolved and will be overwritten if you edit it directly.

## Basic resolv.conf Format

```
# /etc/resolv.conf

# Primary DNS server
nameserver 8.8.8.8

# Secondary DNS server
nameserver 1.1.1.1

# Optional: search domain (appended when resolving short names)
search example.com corp.internal

# Optional: override search/domain order
options ndots:5
```

## Checking the Current DNS Configuration

```bash
# View the current resolv.conf
cat /etc/resolv.conf

# Check if it is managed (look for the generated header)
head -3 /etc/resolv.conf
```

If you see `# Generated by NetworkManager` or `# This is /run/systemd/resolve/stub-resolv.conf`, direct edits will be overwritten.

## Preventing NetworkManager from Overwriting

```bash
# Tell NetworkManager not to manage resolv.conf
sudo tee /etc/NetworkManager/conf.d/no-dns.conf << 'EOF'
[main]
dns=none
EOF

sudo systemctl restart NetworkManager

# Now edit /etc/resolv.conf directly
sudo nano /etc/resolv.conf
```

## Making resolv.conf Immutable

```bash
# Lock the file from being overwritten
sudo chattr +i /etc/resolv.conf

# Verify
lsattr /etc/resolv.conf
# Output: ----i----------- /etc/resolv.conf
```

To edit it again:

```bash
sudo chattr -i /etc/resolv.conf
```

## Using systemd-resolved (Modern Ubuntu)

```bash
# Configure DNS via systemd-resolved
sudo tee /etc/systemd/resolved.conf << 'EOF'
[Resolve]
DNS=8.8.8.8 1.1.1.1
FallbackDNS=9.9.9.9
Domains=example.com
EOF

sudo systemctl restart systemd-resolved

# Verify
resolvectl status
```

The stub resolver at `127.0.0.53` handles queries; `/etc/resolv.conf` should point to it:

```bash
ls -la /etc/resolv.conf
# Should be a symlink to /run/systemd/resolve/stub-resolv.conf
```

## Setting DNS per Interface with NetworkManager

```bash
# Set DNS servers for a specific connection
nmcli con mod "Wired connection 1" ipv4.dns "8.8.8.8,1.1.1.1"
nmcli con up "Wired connection 1"
```

## DNS Search Domains

```bash
# Add search domains so "server" resolves to "server.example.com"
nmcli con mod "Wired connection 1" ipv4.dns-search "example.com,corp.local"
```

Or in `/etc/resolv.conf`:

```
search example.com corp.local
```

## Testing DNS Resolution

```bash
# Test the configured DNS
dig google.com @8.8.8.8
nslookup google.com 8.8.8.8

# Check what resolver is being used
dig +short TXT whoami.resolver.cymru.com
```

## Conclusion

On modern Linux, DNS is best managed through systemd-resolved or NetworkManager rather than editing `/etc/resolv.conf` directly. If direct editing is required, prevent overwrites with `chattr +i` or by disabling DNS management in NetworkManager.
