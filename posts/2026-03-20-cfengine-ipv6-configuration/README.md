# How to Configure IPv6 with CFEngine

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: CFEngine, IPv6, Configuration Management, Policy, Automation

Description: A guide to configuring IPv6 network settings using CFEngine policy language (CFScript/cf3), including sysctl parameters, file management, and command execution for IPv6 setup.

CFEngine is one of the oldest configuration management tools, designed for high scalability and convergence. Its policy language (based on promise theory) is well-suited for maintaining consistent IPv6 configuration across large server fleets.

## CFEngine Policy Structure

```text
/var/cfengine/inputs/
├── promises.cf          # Main policy
├── ipv6/
│   ├── ipv6_sysctl.cf   # Kernel parameters
│   ├── ipv6_firewall.cf # Firewall rules
│   └── ipv6_common.cf   # Shared variables
└── def.cf               # Site definitions
```

## Variables and Classes

```cfengine3
# ipv6/ipv6_common.cf

bundle common ipv6_config {
  vars:
    # IPv6 configuration variables
    "disable_ipv6"        int => "0";
    "enable_forwarding"   int => "0";
    "accept_ra"           int => "1";
    "use_tempaddr"        int => "1";
    "ipv6_prefix"         string => "2001:db8::/32";

  classes:
    # Define class based on role
    "is_router"  expression => "any";
    "is_webserver" expression => "any";
}
```

## sysctl Configuration

```cfengine3
# ipv6/ipv6_sysctl.cf

bundle agent ipv6_sysctl {
  vars:
    "sysctl_settings" slist => {
      "net.ipv6.conf.all.disable_ipv6=0",
      "net.ipv6.conf.default.disable_ipv6=0",
      "net.ipv6.conf.all.accept_ra=1",
      "net.ipv6.conf.all.forwarding=0",
    };

  files:
    # Manage sysctl configuration file
    "/etc/sysctl.d/60-ipv6.conf"
      create => "true",
      edit_line => ipv6_sysctl_content,
      perms => mog("644", "root", "root"),
      classes => if_repaired("sysctl_changed");

  commands:
    sysctl_changed::
      # Reload sysctl when file changes
      "/sbin/sysctl --system"
        comment => "Reload sysctl after IPv6 config change";
}

bundle edit_line ipv6_sysctl_content {
  insert_lines:
    "# IPv6 configuration managed by CFEngine";
    "net.ipv6.conf.all.disable_ipv6 = 0";
    "net.ipv6.conf.default.disable_ipv6 = 0";
    "net.ipv6.conf.all.forwarding = 0";
    "net.ipv6.conf.all.accept_ra = 1";
    "net.ipv6.conf.all.use_tempaddr = 1";
}
```

## File-Based IPv6 Configuration

```cfengine3
# ipv6/ipv6_netplan.cf

bundle agent ipv6_netplan {
  vars:
    "netplan_file" string => "/etc/netplan/99-ipv6-cfengine.yaml";
    "interface" string => "eth0";

  files:
    "/etc/netplan/99-ipv6-cfengine.yaml"
      create => "true",
      edit_template => "$(sys.inputdir)/ipv6/netplan-ipv6.mustache",
      template_method => "mustache",
      template_data => parsejson('{
        "interface": "eth0",
        "accept_ra": true,
        "addresses": []
      }'),
      classes => if_repaired("netplan_changed");

  commands:
    netplan_changed::
      "/usr/sbin/netplan apply"
        comment => "Apply netplan configuration";
}
```

## Command-Based IPv6 Configuration

```cfengine3
# ipv6/ipv6_commands.cf

bundle agent ipv6_kernel_params {
  vars:
    "params" slist => {
      "net.ipv6.conf.all.disable_ipv6",
      "net.ipv6.conf.all.accept_ra",
    };

  commands:
    # Set IPv6 parameters via sysctl command
    "/sbin/sysctl -w net.ipv6.conf.all.disable_ipv6=0"
      if => not(returnszero("/sbin/sysctl net.ipv6.conf.all.disable_ipv6 | grep -q '= 0'", "bash"));

    "/sbin/sysctl -w net.ipv6.conf.all.accept_ra=1"
      if => not(returnszero("/sbin/sysctl net.ipv6.conf.all.accept_ra | grep -q '= 1'", "bash"));
}
```

## ip6tables Management

```cfengine3
# ipv6/ipv6_firewall.cf

bundle agent ipv6_firewall {
  vars:
    "rules_file" string => "/etc/ip6tables.rules";

  files:
    "/etc/ip6tables.rules"
      create => "true",
      edit_line => ip6tables_rules,
      perms => mog("600", "root", "root"),
      classes => if_repaired("ip6tables_changed");

  commands:
    ip6tables_changed::
      "/sbin/ip6tables-restore < /etc/ip6tables.rules"
        comment => "Load IPv6 firewall rules";
}

bundle edit_line ip6tables_rules {
  insert_lines:
    "# Generated by CFEngine";
    "*filter";
    ":INPUT DROP [0:0]";
    ":FORWARD DROP [0:0]";
    ":OUTPUT ACCEPT [0:0]";
    "-A INPUT -i lo -j ACCEPT";
    "-A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT";
    "-A INPUT -p ipv6-icmp -j ACCEPT";
    "-A INPUT -p tcp --dport 22 -j ACCEPT";
    "COMMIT";
}
```

## Running CFEngine Policy

```bash
# Test policy syntax

cf-promises -f /var/cfengine/inputs/promises.cf

# Run agent (apply policy)
cf-agent -f /var/cfengine/inputs/promises.cf -b ipv6_sysctl

# Run with verbose output
cf-agent -v -f /var/cfengine/inputs/promises.cf

# Schedule via cf-execd (runs every 5 minutes by default)
systemctl start cfengine3
systemctl enable cfengine3
```

CFEngine's convergence model - continuously checking and correcting configuration drift - makes it effective for maintaining IPv6 settings across long-lived server fleets where manual changes might otherwise cause configuration drift over time.
