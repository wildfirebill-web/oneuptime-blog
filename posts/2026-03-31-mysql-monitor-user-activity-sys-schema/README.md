# How to Monitor User Activity with sys Schema in MySQL

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: MySQL, sys Schema, User, Activity, Monitoring

Description: Use MySQL sys schema user_summary views to monitor per-user query counts, latency, I/O activity, and memory usage for security and performance auditing.

---

## Overview

In multi-tenant applications or databases accessed by multiple services, per-user activity monitoring helps identify which database users consume the most resources, generate the most errors, or exhibit unusual access patterns. The `sys` schema `user_summary` views make this straightforward.

## Using user_summary

The main `user_summary` view gives a per-user overview:

```sql
SELECT
  user,
  statements,
  statement_latency,
  table_scans,
  file_ios,
  file_io_latency,
  current_connections,
  current_memory
FROM sys.user_summary
ORDER BY statements DESC;
```

## user_summary_by_statement_latency

Find users running the most resource-intensive queries:

```sql
SELECT
  user,
  total,
  total_latency,
  avg_latency,
  max_latency,
  lock_latency
FROM sys.user_summary_by_statement_latency
ORDER BY total_latency DESC;
```

## user_summary_by_statement_type

Understand what types of SQL each user is executing:

```sql
SELECT
  user,
  statement,
  total,
  total_latency,
  rows_sent,
  rows_examined
FROM sys.user_summary_by_statement_type
WHERE user = 'app_user'
ORDER BY total_latency DESC;
```

## user_summary_by_file_io

Monitor disk I/O generated by each user:

```sql
SELECT
  user,
  ios,
  io_latency
FROM sys.user_summary_by_file_io
ORDER BY io_latency DESC;
```

## user_summary_by_stages

Identify users whose queries spend time in expensive stages:

```sql
SELECT
  user,
  event_name,
  total,
  total_latency,
  avg_latency
FROM sys.user_summary_by_stages
ORDER BY total_latency DESC
LIMIT 20;
```

## Querying Performance Schema Directly

```sql
SELECT
  USER,
  CURRENT_CONNECTIONS,
  TOTAL_CONNECTIONS
FROM performance_schema.users
WHERE USER IS NOT NULL
ORDER BY CURRENT_CONNECTIONS DESC;
```

## Combining User Stats with Errors

```sql
SELECT
  u.user,
  u.statements,
  u.statement_latency,
  (SELECT SUM_ERRORS
   FROM performance_schema.events_statements_summary_by_user_by_event_name
   WHERE USER = u.user
   LIMIT 1) AS total_errors
FROM sys.user_summary u
ORDER BY total_errors DESC;
```

## Security Monitoring with User Activity

```sql
-- Find users with unusual file I/O (potential SELECT INTO OUTFILE misuse)
SELECT user, ios, io_latency
FROM sys.user_summary_by_file_io
WHERE user NOT IN ('mysql.sys', 'root')
ORDER BY ios DESC;
```

## Summary

The `sys.user_summary` views expose per-database-user performance metrics including statement counts, latency, I/O, and memory usage. Regular monitoring of user activity helps identify resource-heavy service accounts, detect anomalous query patterns, and enforce performance budgets per application tier in shared MySQL environments.
