# How to Convert Docker Compose to Kubernetes Manifests with Portainer Terraform

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Portainer, Terraform, Kubernetes, Docker Compose, Migration

Description: Learn strategies for migrating Docker Compose stacks to Kubernetes manifests and managing them through Portainer with Terraform.

## The Migration Challenge

Moving from Docker Compose (managed by Portainer stacks) to Kubernetes (managed by Portainer k8s environments) requires converting your Compose definitions to Kubernetes manifests. This guide covers the process and how to manage the result with Terraform.

## Tool: Kompose

Kompose is the official tool for converting Docker Compose files to Kubernetes manifests:

```bash
# Install Kompose

curl -L https://github.com/kubernetes/kompose/releases/download/v1.31.2/kompose-linux-amd64 \
  -o kompose
chmod +x kompose

# Convert a docker-compose.yml to Kubernetes manifests
kompose convert -f docker-compose.yml -o ./k8s/

# View generated files
ls k8s/
# my-app-deployment.yaml
# my-app-service.yaml
# postgres-deployment.yaml
# postgres-service.yaml
# postgres-persistentvolumeclaim.yaml
```

## Example: Docker Compose Input

```yaml
# docker-compose.yml (input)
version: "3.8"

services:
  web:
    image: registry.mycompany.com/web:1.5.0
    ports:
      - "80:8080"
    environment:
      - DB_HOST=postgres
      - DB_PORT=5432
    depends_on:
      - postgres

  postgres:
    image: postgres:15-alpine
    environment:
      - POSTGRES_DB=myapp
      - POSTGRES_PASSWORD=secret
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:
```

## Generated Kubernetes Manifests

```bash
kompose convert -f docker-compose.yml -o k8s/

# k8s/web-deployment.yaml (generated)
```

```yaml
# Generated by Kompose - review and customize before use
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web
spec:
  replicas: 1
  selector:
    matchLabels:
      io.kompose.service: web
  template:
    spec:
      containers:
        - name: web
          image: registry.mycompany.com/web:1.5.0
          ports:
            - containerPort: 8080
          env:
            - name: DB_HOST
              value: postgres
            - name: DB_PORT
              value: "5432"
```

## Deploying Converted Manifests via Portainer Terraform

After converting and customizing the manifests, deploy them to a Kubernetes environment:

```hcl
# k8s-stacks.tf
# Deploy the converted Kubernetes manifests via Portainer

locals {
  k8s_manifests = join("---\n", [
    file("${path.module}/k8s/web-deployment.yaml"),
    file("${path.module}/k8s/web-service.yaml"),
    file("${path.module}/k8s/postgres-deployment.yaml"),
    file("${path.module}/k8s/postgres-service.yaml"),
    file("${path.module}/k8s/postgres-persistentvolumeclaim.yaml"),
  ])
}

resource "portainer_kubernetes_manifest" "app" {
  endpoint_id = portainer_environment.k8s_prod.id
  namespace   = "production"
  manifest    = local.k8s_manifests
}
```

## Post-Migration Improvements

After converting, enhance the manifests with Kubernetes-specific features:

```yaml
# Add resource limits (not in Compose)
resources:
  requests:
    cpu: 100m
    memory: 128Mi
  limits:
    cpu: 500m
    memory: 512Mi

# Add health checks
readinessProbe:
  httpGet:
    path: /health
    port: 8080
  initialDelaySeconds: 10

# Use Kubernetes Secrets instead of plain env vars
env:
  - name: DB_PASSWORD
    valueFrom:
      secretKeyRef:
        name: db-secrets
        key: password
```

## Conclusion

Migrating from Docker Compose to Kubernetes manifests is a straightforward process with Kompose. After conversion, use the Portainer Terraform provider to deploy and manage the resulting Kubernetes resources through your GitOps pipeline.
