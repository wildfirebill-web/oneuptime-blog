# How to Use SHOW WARNINGS in MySQL

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: MySQL, SHOW WARNINGS, Debugging, SQL Diagnostics

Description: Learn how to use SHOW WARNINGS in MySQL to retrieve error, warning, and note messages generated by the previous SQL statement.

---

## What Is SHOW WARNINGS in MySQL

`SHOW WARNINGS` displays the error, warning, and note messages produced by the most recently executed statement in the current session. MySQL generates warnings for issues that did not cause the statement to fail but may indicate data truncation, implicit type conversion, deprecated syntax, or other noteworthy conditions.

## Basic Syntax

```sql
SHOW WARNINGS;
SHOW WARNINGS LIMIT n;
SHOW WARNINGS LIMIT offset, row_count;
```

`SHOW ERRORS` is a related command that shows only errors (not notes or warnings).

## Viewing Warnings After an INSERT

When MySQL silently modifies data during an INSERT, it generates a warning:

```sql
CREATE TABLE demo (name VARCHAR(5));
INSERT INTO demo VALUES ('Hello World');  -- truncated to 5 chars
SHOW WARNINGS;
```

```text
+---------+------+------------------------------------------+
| Level   | Code | Message                                  |
+---------+------+------------------------------------------+
| Warning | 1265 | Data truncated for column 'name' at row 1|
+---------+------+------------------------------------------+
```

The three columns are:

- `Level` - Error, Warning, or Note
- `Code` - MySQL error code
- `Message` - human-readable description

## Warnings After Data Type Conversion

```sql
CREATE TABLE orders (total DECIMAL(10,2));
INSERT INTO orders VALUES ('$123.45');
SHOW WARNINGS;
```

```text
+---------+------+---------------------------------------------+
| Level   | Code | Message                                     |
+---------+------+---------------------------------------------+
| Warning | 1292 | Incorrect decimal value: '$123.45'          |
+---------+------+---------------------------------------------+
```

## Using SHOW WARNINGS with EXPLAIN

After running `EXPLAIN` on a complex query, check for notes about optimizer decisions:

```sql
EXPLAIN SELECT * FROM orders WHERE YEAR(created_at) = 2026;
SHOW WARNINGS\G
```

```text
*************************** 1. row ***************************
  Level: Note
   Code: 1003
Message: /* select#1 */ select `mydb`.`orders`.`id` AS `id` ...
```

This shows the optimized form of the query that MySQL actually executed.

## Checking Warning Count Without Fetching Details

```sql
-- After any statement, the warning count is available
INSERT INTO demo VALUES ('Overflow');
SELECT @@warning_count;
-- or
SHOW COUNT(*) WARNINGS;
```

## Automatically Displaying Warnings in mysql Client

Use `\W` to enable automatic warning display after every statement:

```sql
\W
-- Now every statement automatically shows SHOW WARNINGS output
```

Disable with `\w`.

## Warnings from ALTER TABLE

Schema changes often produce notes about implicit behavior:

```sql
ALTER TABLE users MODIFY COLUMN age TINYINT UNSIGNED;
SHOW WARNINGS;
```

```text
+-------+------+----------------------------------------------------------+
| Level | Code | Message                                                  |
+-------+------+----------------------------------------------------------+
| Note  | 1265 | Data truncated for column 'age' at row 5                 |
+-------+------+----------------------------------------------------------+
```

## Common Warning Codes

| Code | Meaning |
|---|---|
| 1265 | Data truncated for column |
| 1292 | Incorrect value for column |
| 1366 | Incorrect integer value |
| 1364 | Field has no default value |
| 1048 | Column cannot be null |
| 1062 | Duplicate entry (this escalates to error) |

## Setting sql_mode to Treat Warnings as Errors

To force MySQL to raise an error on data truncation instead of silently truncating:

```sql
SET sql_mode = 'STRICT_ALL_TABLES';
-- or
SET sql_mode = 'STRICT_TRANS_TABLES';
```

With strict mode enabled, the INSERT that produced a truncation warning will now fail with an error instead.

## Warnings in Application Code

In Python, check for warnings programmatically:

```python
import mysql.connector
import warnings

conn = mysql.connector.connect(host='localhost', database='mydb',
                               user='root', password='secret')
cursor = conn.cursor()
cursor.execute("INSERT INTO demo VALUES ('Hello World')")

cursor.execute("SHOW WARNINGS")
for level, code, message in cursor.fetchall():
    print(f"[{level}] {code}: {message}")

conn.close()
```

## Summary

`SHOW WARNINGS` reveals silent issues that MySQL tolerated without raising an error, such as data truncation, type conversion, and deprecated syntax usage. Always check warnings after DDL changes and bulk data operations. Use `STRICT_ALL_TABLES` or `STRICT_TRANS_TABLES` SQL modes in production to convert dangerous warnings into hard errors that protect data integrity.
