# How to Use Helm Charts for MongoDB Deployment

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: MongoDB, Helm, Kubernetes, Chart, Deployment

Description: Learn how to deploy MongoDB on Kubernetes using the Bitnami Helm chart, including configuring replica sets, persistent storage, authentication, and custom values.

---

## Overview

Helm simplifies deploying MongoDB on Kubernetes by packaging all required manifests into a chart with sensible defaults that you can override with a `values.yaml` file. The Bitnami MongoDB Helm chart is the most widely used and supports standalone, replica set, and sharded configurations.

## Adding the Bitnami Helm Repository

```bash
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update
```

## Deploying a Standalone MongoDB Instance

For development or testing:

```bash
helm install my-mongodb bitnami/mongodb \
  --namespace mongodb \
  --create-namespace \
  --set auth.rootPassword=securepassword \
  --set persistence.size=10Gi
```

## Deploying a Replica Set

Create a `values.yaml` file for a production replica set:

```yaml
architecture: replicaset

auth:
  rootPassword: "securepassword"
  replicaSetKey: "replicaSetKeySecret"

replicaCount: 3

persistence:
  enabled: true
  size: 20Gi
  storageClass: "standard"

resources:
  requests:
    memory: "1Gi"
    cpu: "500m"
  limits:
    memory: "2Gi"
    cpu: "1000m"

metrics:
  enabled: true
  serviceMonitor:
    enabled: true
    namespace: monitoring
```

Deploy with the values file:

```bash
helm install my-mongodb bitnami/mongodb \
  --namespace mongodb \
  --create-namespace \
  -f values.yaml
```

## Connecting to the Deployed MongoDB

```bash
# Get the root password
export MONGODB_ROOT_PASSWORD=$(kubectl get secret my-mongodb \
  -n mongodb -o jsonpath="{.data.mongodb-root-password}" | base64 --decode)

# Connect via mongosh
kubectl exec -it my-mongodb-0 -n mongodb -- \
  mongosh -u root -p "$MONGODB_ROOT_PASSWORD" --authenticationDatabase admin
```

## Upgrading the Chart

```bash
helm upgrade my-mongodb bitnami/mongodb \
  --namespace mongodb \
  -f values.yaml
```

## Enabling TLS

```yaml
tls:
  enabled: true
  autoGenerated: true
```

## Uninstalling and Cleanup

```bash
helm uninstall my-mongodb -n mongodb

# Delete persistent volume claims if needed
kubectl delete pvc -n mongodb -l app.kubernetes.io/instance=my-mongodb
```

## Checking Deployed Resources

```bash
helm status my-mongodb -n mongodb
helm get values my-mongodb -n mongodb
kubectl get all -n mongodb -l app.kubernetes.io/instance=my-mongodb
```

## Summary

The Bitnami MongoDB Helm chart lets you deploy a replica set with a single command and a `values.yaml` file. Configure authentication, persistent storage, resource limits, and metrics by setting values in the YAML file. Use `helm upgrade` to apply configuration changes or update to newer MongoDB versions without rebuilding manifests from scratch.
