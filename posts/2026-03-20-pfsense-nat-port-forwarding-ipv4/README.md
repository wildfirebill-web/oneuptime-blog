# How to Set Up NAT Port Forwarding on pfSense for IPv4

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: pfSense, NAT, Port Forwarding, IPv4, Firewall, DNAT

Description: Configure NAT port forwarding (destination NAT) on pfSense to forward incoming IPv4 connections on specific ports to internal servers, with automatic firewall rule creation.

## Introduction

pfSense NAT port forwarding redirects incoming connections on the WAN IP to an internal server. The GUI wizard creates both the NAT rule and the associated firewall allow rule simultaneously.

## Port Forwarding via GUI

Navigate to **Firewall > NAT > Port Forward > Add**:

```text
Interface:       WAN
Protocol:        TCP
Destination:     WAN address
Destination port range: 80 (HTTP)
Redirect target IP:    192.168.1.100
Redirect target port:  80
Description:     Web server port forward

Filter rule association: Add associated filter rule  ← Important
```

## Common Port Forwarding Examples

| Service | External Port | Internal IP | Internal Port |
|---------|--------------|-------------|---------------|
| HTTP | 80 | 192.168.1.100 | 80 |
| HTTPS | 443 | 192.168.1.100 | 443 |
| SSH | 2222 | 192.168.1.200 | 22 |
| RDP | 3389 | 192.168.1.50 | 3389 |
| Minecraft | 25565 | 192.168.1.30 | 25565 |

## pfSense pf Rules (Behind the Scenes)

```pf
# /tmp/rules.debug (generated by pfSense)

# Port forward: HTTP to web server

rdr pass on em0 proto tcp from any to (em0) port 80 \
    -> 192.168.1.100 port 80

# Associated filter rule
pass in on em0 proto tcp from any to 192.168.1.100 port 80 \
    flags S/SA keep state
```

## Port Range Forward (Game Server)

Navigate to **Firewall > NAT > Port Forward > Add**:
```text
Protocol:               UDP
Destination port range: 27015 to 27020
Redirect target IP:     192.168.1.30
Redirect target port:   27015
```

## NAT Reflection (Hairpin NAT)

When LAN hosts need to access a server via its public IP:

Navigate to **Firewall > NAT > Outbound**:
- Mode: **Hybrid Outbound NAT**

Then in the port forward rule:
- NAT reflection: **Enable (Pure NAT)**

Or globally: **System > Advanced > Firewall & NAT**:
- NAT Reflection mode for port forwards: **Pure NAT**

## Verify Port Forwards

```bash
# From pfSense shell (Diagnostics > Command Prompt)
# Test if rule is active
pfctl -s nat

# Check state table
pfctl -s state | grep 192.168.1.100

# External test
curl http://<WAN-IP>/
nc -zv <WAN-IP> 80
```

## Conclusion

pfSense port forwarding is configured under **Firewall > NAT > Port Forward**. Always enable **Filter rule association** so the corresponding firewall allow rule is created automatically. Enable NAT Reflection if LAN clients need to reach forwarded ports using the WAN IP address.
