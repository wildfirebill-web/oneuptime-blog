# How to Build a Vector Similarity Search with ClickHouse

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: ClickHouse, Vector Search, Similarity Search, Embedding, Annoy Index, Machine Learning

Description: Learn how to build a vector similarity search system in ClickHouse using ANN indexes and distance functions for semantic search and recommendations.

---

Vector similarity search enables finding the most semantically similar items in a dataset - a core requirement for recommendation engines, semantic search, and AI-powered applications. ClickHouse supports vector operations natively through its distance functions and approximate nearest neighbor (ANN) indexes.

## What Is Vector Similarity Search

A vector similarity search takes a query vector (an embedding) and finds the N closest vectors in your dataset based on a distance metric such as cosine similarity or Euclidean distance. In practice, embeddings are generated by ML models (e.g., OpenAI, Sentence Transformers) and stored alongside the original records.

## Storing Embeddings in ClickHouse

Use the `Array(Float32)` data type to store fixed-length embedding vectors:

```sql
CREATE TABLE product_embeddings
(
    product_id  UInt64,
    title       String,
    description String,
    embedding   Array(Float32),
    created_at  DateTime DEFAULT now()
)
ENGINE = MergeTree()
ORDER BY product_id;
```

Insert sample data:

```sql
INSERT INTO product_embeddings (product_id, title, description, embedding) VALUES
(1, 'Wireless Headphones', 'Over-ear noise-cancelling', [0.12, 0.87, 0.34, 0.56]),
(2, 'Bluetooth Speaker',   'Portable waterproof speaker', [0.15, 0.82, 0.31, 0.60]),
(3, 'USB-C Cable',         '2m braided cable',            [0.91, 0.10, 0.05, 0.20]);
```

## Running Similarity Queries with Distance Functions

ClickHouse provides several built-in distance functions:

```sql
-- Find top 5 products most similar to a query vector using cosine distance
SELECT
    product_id,
    title,
    cosineDistance(embedding, [0.13, 0.85, 0.33, 0.58]) AS dist
FROM product_embeddings
ORDER BY dist ASC
LIMIT 5;
```

For Euclidean (L2) distance:

```sql
SELECT
    product_id,
    title,
    L2Distance(embedding, [0.13, 0.85, 0.33, 0.58]) AS dist
FROM product_embeddings
ORDER BY dist ASC
LIMIT 5;
```

## Adding an ANN Index for Faster Lookups

For large datasets, a full scan is too slow. Add an approximate nearest neighbor (ANN) index using the `usearch` or `annoy` engine:

```sql
ALTER TABLE product_embeddings
    ADD INDEX embedding_ann_idx embedding
    TYPE usearch('cosineDistance')
    GRANULARITY 100000000;

ALTER TABLE product_embeddings MATERIALIZE INDEX embedding_ann_idx;
```

Query using the index hint:

```sql
SELECT
    product_id,
    title,
    cosineDistance(embedding, [0.13, 0.85, 0.33, 0.58]) AS dist
FROM product_embeddings
WHERE cosineDistance(embedding, [0.13, 0.85, 0.33, 0.58]) < 0.2
ORDER BY dist ASC
LIMIT 10
SETTINGS ann_index_select_query_params = '{"ef": 128}';
```

## Combining Metadata Filters with Vector Search

A key advantage of ClickHouse is combining structured filters with ANN queries:

```sql
SELECT
    product_id,
    title,
    cosineDistance(embedding, [0.13, 0.85, 0.33, 0.58]) AS dist
FROM product_embeddings
WHERE created_at >= '2024-01-01'
ORDER BY dist ASC
LIMIT 10;
```

## Performance Tips

- Normalize embeddings before insertion when using cosine distance to improve index quality.
- Keep embedding dimensions consistent - changing them requires rebuilding the table.
- Partition tables by date for time-scoped similarity searches.
- Benchmark `usearch` vs `annoy` index types for your specific workload.

## Summary

ClickHouse supports vector similarity search through native distance functions (`cosineDistance`, `L2Distance`) and ANN indexes. You can store embeddings as `Array(Float32)`, index them with `usearch` or `annoy` index types, and combine similarity queries with standard SQL filters for powerful hybrid search pipelines.
