# How to Deploy a Kubernetes Application via Form in Portainer

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Portainer, Kubernetes, Application Deployment, UI, DevOps

Description: Learn how to deploy a Kubernetes application using Portainer's guided form interface without writing YAML manifests.

## Overview

Portainer provides a form-based deployment interface for Kubernetes that abstracts away YAML complexity. You fill in fields for image, replicas, environment variables, and networking, and Portainer generates and applies the manifests automatically.

## Deploying an Application

1. Select your Kubernetes environment in Portainer.
2. Go to **Applications** in the sidebar.
3. Click **Add application**.
4. Fill in the form fields.

## Form Fields Explained

### Basic Settings

- **Name**: Application name (used as Deployment and Service name).
- **Namespace**: Select the target namespace.
- **Registry**: Choose the registry and image name.
- **Image**: Full image reference (e.g., `nginx:1.25-alpine`).

### Deployment Configuration

- **Replicas**: Number of pod replicas.
- **Deployment type**: Rolling update (Deployment), StatefulSet, or DaemonSet.

### Networking

- **Service type**: None, ClusterIP, NodePort, or LoadBalancer.
- **Port mappings**: Map container ports to service ports.

### Environment Variables

Add key-value pairs directly in the form. Portainer injects these as `env` entries in the pod spec:

```yaml
# Equivalent YAML generated by Portainer for environment variables
env:
  - name: DB_HOST
    value: "postgres.default.svc.cluster.local"
  - name: APP_ENV
    value: "production"
```

### Persistent Storage

- **Volume name**: A name for the volume.
- **Mount path**: Where to mount in the container.
- **Storage class**: Select the Kubernetes storage class.
- **Size**: Requested storage size.

## Example: Deploying Nginx via Form

Fill in the form as follows:
- **Name**: `web-nginx`
- **Namespace**: `default`
- **Image**: `nginx:1.25-alpine`
- **Replicas**: `2`
- **Service type**: `NodePort`
- **Container port**: `80`, **Service port**: `80`

Portainer generates and applies this Deployment and Service:

```yaml
# Auto-generated Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-nginx
  namespace: default
spec:
  replicas: 2
  selector:
    matchLabels:
      app: web-nginx
  template:
    spec:
      containers:
        - name: web-nginx
          image: nginx:1.25-alpine
          ports:
            - containerPort: 80
```

## After Deployment

Once deployed, the application appears in **Applications** with:
- Status (Running/Pending/Failed)
- Pod count
- Access URL (if a NodePort or LoadBalancer service was created)

Click the application name to view logs, shell into pods, scale replicas, or edit the configuration.

## Conclusion

Portainer's form-based deployment interface lowers the barrier to deploying Kubernetes applications. It's particularly valuable for teams transitioning from Docker Compose who are new to Kubernetes YAML syntax.
