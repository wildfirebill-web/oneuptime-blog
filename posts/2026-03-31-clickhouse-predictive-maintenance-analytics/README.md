## Introduction

Predictive maintenance (PdM) in manufacturing uses sensor data to predict failures before they occur. ClickHouse stores the large volumes of time-series sensor data generated by machines, and its analytical SQL capabilities enable trend scoring, anomaly detection, and failure correlation without needing a separate ML infrastructure for basic patterns.

## Schema

```sql
CREATE TABLE sensor_readings
(
    asset_id        UInt32,
    asset_class     LowCardinality(String),
    plant_id        UInt16,
    recorded_at     DateTime,
    vibration_g     Float32,
    temperature_c   Float32,
    current_a       Float32,
    pressure_bar    Float32,
    rpm             Float32
)
ENGINE = MergeTree()
PARTITION BY toYYYYMM(recorded_at)
ORDER BY (asset_id, recorded_at)
TTL recorded_at + INTERVAL 7 YEAR;

CREATE TABLE work_orders
(
    asset_id      UInt32,
    opened_at     DateTime,
    closed_at     Nullable(DateTime),
    wo_type       LowCardinality(String),
    failure_mode  LowCardinality(String)
)
ENGINE = MergeTree()
ORDER BY (asset_id, opened_at);
```

## Exponential Moving Average for Vibration Trend

Approximate an EMA to smooth noise and expose trend:

```sql
SELECT
    asset_id,
    toDate(recorded_at) AS day,
    avg(vibration_g)    AS daily_avg_g,
    avg(avg(vibration_g)) OVER (
        PARTITION BY asset_id
        ORDER BY toDate(recorded_at)
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ) AS ema_7d
FROM sensor_readings
WHERE recorded_at >= today() - INTERVAL 60 DAY
GROUP BY asset_id, day
ORDER BY asset_id, day;
```

## Z-Score Anomaly Scoring

```sql
WITH baseline AS (
    SELECT
        asset_id,
        avg(vibration_g)      AS mean_vib,
        stddevPop(vibration_g) AS std_vib,
        avg(temperature_c)    AS mean_temp,
        stddevPop(temperature_c) AS std_temp
    FROM sensor_readings
    WHERE recorded_at BETWEEN today() - 60 AND today() - 8
    GROUP BY asset_id
)
SELECT
    s.asset_id,
    s.recorded_at,
    round(abs(s.vibration_g - b.mean_vib) / b.std_vib, 2)    AS vib_zscore,
    round(abs(s.temperature_c - b.mean_temp) / b.std_temp, 2) AS temp_zscore,
    round(
        (abs(s.vibration_g - b.mean_vib) / b.std_vib +
         abs(s.temperature_c - b.mean_temp) / b.std_temp) / 2,
        2
    ) AS composite_score
FROM sensor_readings s
JOIN baseline b ON s.asset_id = b.asset_id
WHERE s.recorded_at >= today() - INTERVAL 7 DAY
  AND composite_score > 2.5
ORDER BY composite_score DESC
LIMIT 50;
```

## Time to Failure Analysis

Calculate how far in advance sensor anomalies appeared before each failure:

```sql
SELECT
    w.asset_id,
    w.failure_mode,
    s.recorded_at                               AS anomaly_time,
    w.opened_at                                 AS failure_time,
    dateDiff('hour', s.recorded_at, w.opened_at) AS hours_before_failure,
    avg(s.vibration_g)                          AS avg_vib_before
FROM work_orders w
JOIN sensor_readings s
    ON s.asset_id = w.asset_id
   AND s.recorded_at BETWEEN w.opened_at - INTERVAL 72 HOUR AND w.opened_at
WHERE w.wo_type = 'corrective'
  AND s.vibration_g > (SELECT avg(vibration_g) * 1.5 FROM sensor_readings WHERE asset_id = w.asset_id)
GROUP BY w.asset_id, w.failure_mode, s.recorded_at, w.opened_at
ORDER BY hours_before_failure;
```

## Asset Risk Ranking

```sql
SELECT
    asset_id,
    asset_class,
    round(avg(vibration_g), 3)                      AS avg_vib,
    round(max(vibration_g), 3)                      AS max_vib,
    round(avg(temperature_c), 1)                    AS avg_temp,
    countIf(vibration_g > 5.0)                      AS vib_exceedances,
    countIf(temperature_c > 90)                     AS temp_exceedances,
    vib_exceedances + temp_exceedances * 2           AS risk_score
FROM sensor_readings
WHERE recorded_at >= today() - INTERVAL 30 DAY
GROUP BY asset_id, asset_class
ORDER BY risk_score DESC
LIMIT 20;
```

## Summary

ClickHouse is a practical platform for predictive maintenance analytics, providing storage for years of sensor history and SQL-based anomaly scoring, trend detection, and failure correlation. While ML models add sophistication, rule-based z-score and trend queries in ClickHouse can deliver significant value with minimal infrastructure overhead.
