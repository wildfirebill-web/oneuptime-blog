# How to Assign the Administrator Role in Portainer

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Portainer, Administrator, RBAC, User Management, Access Control

Description: Grant administrator privileges to users in Portainer via the web UI and API, and understand the implications of the admin role.

## Introduction

The Administrator role in Portainer provides full, unrestricted access to all environments, settings, and management functions. This guide covers how to grant admin privileges to users, manage admin accounts, and apply the principle of least privilege when working with admin roles.

## What the Administrator Role Enables

An Administrator in Portainer can:

- Access **all environments** regardless of team membership or access policies
- Manage users, teams, and their permissions
- Configure global settings including authentication, registries, and templates
- Add, modify, and remove environments
- Manage the Portainer license (Business Edition)
- Deploy to any environment and manage all containers, stacks, and services
- Access all audit logs and activity history (Business Edition)
- Configure edge computing settings

## Assigning Admin Role via Web UI

### To a New User

1. Navigate to **Settings** → **Users**
2. Click **Add user**
3. Fill in username and password
4. Set **Role** to **Administrator**
5. Click **Create user**

### To an Existing User

1. Navigate to **Settings** → **Users**
2. Click on the user's name
3. Change the **Role** dropdown to **Administrator**
4. Save

## Assigning Admin Role via API

```bash
# Get admin token
TOKEN=$(curl -s -X POST \
  https://portainer.example.com/api/auth \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"adminpassword"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['jwt'])")

# Create a new admin user
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  https://portainer.example.com/api/users \
  -d '{
    "username": "alice-admin",
    "password": "SecureAdminP@ssword123!",
    "role": 1
  }'

# Promote an existing user (user ID: 5) to admin
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  https://portainer.example.com/api/users/5 \
  -d '{
    "role": 1
  }'

# Demote an admin (user ID: 5) to standard user
curl -X PUT \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  https://portainer.example.com/api/users/5 \
  -d '{
    "role": 2
  }'
```

## Listing All Administrators

```bash
# Get all users with admin role (role: 1)
curl -s \
  -H "Authorization: Bearer $TOKEN" \
  https://portainer.example.com/api/users \
  | python3 -c "
import sys, json
users = json.load(sys.stdin)
admins = [u for u in users if u.get('Role') == 1]
print('Administrators:')
for admin in admins:
    print(f'  ID: {admin[\"Id\"]}, Username: {admin[\"Username\"]}')
"
```

## Best Practices for Admin Role Management

### Minimum Admin Accounts

Keep the number of admin accounts as small as possible:

| Organization Size | Recommended Admin Count |
|------------------|------------------------|
| Small (1-10 people) | 1-2 admins |
| Medium (11-50 people) | 2-3 admins |
| Large (50+ people) | 3-5 admins (with documented handoff) |

### Use Service Accounts for Automation

For API automation that needs admin access, create a dedicated service account:

```bash
# Create admin service account for CI/CD
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  https://portainer.example.com/api/users \
  -d '{
    "username": "ci-cd-service-account",
    "password": "AutoGeneratedRandomPassword",
    "role": 1
  }'

# Use API tokens for the service account (not password)
```

### Audit Admin Activity

In Portainer Business Edition, audit logs track admin actions:

```bash
curl -s \
  -H "Authorization: Bearer $TOKEN" \
  "https://portainer.example.com/api/audit?role=1" \
  | python3 -m json.tool
```

## Preventing Admin Lock-Out

Always maintain at least two admin accounts. If you're the only admin and forget your password:

```bash
# Use the helper container to reset admin password
docker stop portainer
docker run --rm -v portainer_data:/data \
  portainer/helper-reset-password
docker start portainer
```

## Conclusion

The Administrator role is powerful and should be assigned carefully. Create admin accounts only for users who genuinely need global access, use dedicated service accounts for automation, and maintain at least two admin accounts for redundancy. For most operational staff, Standard User or Operator roles with environment-specific access is more appropriate than full admin privileges.
