# How to Create AWS IoT Core Things with OpenTofu

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: OpenTofu, AWS, IoT Core, IoT, Device Management, Infrastructure as Code

Description: Learn how to create AWS IoT Core Things, thing types, thing groups, and certificates for IoT device management using OpenTofu.

## Introduction

AWS IoT Core Things represent physical devices in the cloud registry. Each thing can have attributes, be assigned to groups, and associated with certificates for authentication. OpenTofu manages the device registry infrastructure as code.

## Creating a Thing Type

```hcl
resource "aws_iot_thing_type" "sensor" {
  name = "temperature-sensor"

  properties {
    description           = "Temperature and humidity sensor device"
    searchable_attributes = ["location", "floor", "building"]
  }

  tags = {
    Environment = var.environment
    ManagedBy   = "opentofu"
  }
}
```

## Creating a Thing Group

```hcl
resource "aws_iot_thing_group" "building_a" {
  name = "building-a-sensors"

  properties {
    description = "All sensors in Building A"
    attribute_payload {
      attributes = {
        building = "A"
        city     = "Seattle"
      }
    }
  }

  tags = {
    Environment = var.environment
    ManagedBy   = "opentofu"
  }
}
```

## Creating a Thing

```hcl
resource "aws_iot_thing" "sensor_001" {
  name            = "sensor-001"
  thing_type_name = aws_iot_thing_type.sensor.name

  attributes = {
    location  = "floor-3-room-301"
    floor     = "3"
    building  = "A"
    installed = "2026-01-15"
  }
}
```

## Adding Thing to a Group

```hcl
resource "aws_iot_thing_group_membership" "sensor_001" {
  thing_name       = aws_iot_thing.sensor_001.name
  thing_group_name = aws_iot_thing_group.building_a.name
}
```

## Creating a Certificate and Attaching to a Thing

```hcl
# Create a certificate (key and cert are generated by AWS)

resource "aws_iot_certificate" "sensor_001" {
  active = true
}

# Attach policy to certificate
resource "aws_iot_policy_attachment" "sensor_001" {
  policy = aws_iot_policy.device_policy.name
  target = aws_iot_certificate.sensor_001.arn
}

# Attach certificate to thing
resource "aws_iot_thing_principal_attachment" "sensor_001" {
  principal = aws_iot_certificate.sensor_001.arn
  thing     = aws_iot_thing.sensor_001.name
}
```

## Bulk Thing Creation

Use `count` or `for_each` to provision multiple things.

```hcl
variable "device_ids" {
  type    = list(string)
  default = ["sensor-001", "sensor-002", "sensor-003"]
}

resource "aws_iot_thing" "sensors" {
  for_each        = toset(var.device_ids)
  name            = each.key
  thing_type_name = aws_iot_thing_type.sensor.name
}
```

## Outputs

```hcl
output "thing_name" {
  value = aws_iot_thing.sensor_001.name
}

output "certificate_pem" {
  description = "Certificate PEM – store securely on the device"
  value       = aws_iot_certificate.sensor_001.certificate_pem
  sensitive   = true
}

output "private_key" {
  description = "Private key – store securely on the device"
  value       = aws_iot_certificate.sensor_001.private_key
  sensitive   = true
}
```

## Deploying

```bash
tofu init
tofu plan -out=tfplan
tofu apply tfplan
```

## Summary

AWS IoT Core Things provide a cloud registry for your physical devices. OpenTofu manages thing types, groups, individual things, and their certificate associations - enabling fleet provisioning and device lifecycle management as code.
