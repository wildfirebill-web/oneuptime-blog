# How to Configure Kubernetes Application Environment Variables in Portainer

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Portainer, Kubernetes, Environment Variables, ConfigMap, Secrets

Description: Learn how to add and manage environment variables for Kubernetes applications deployed through Portainer.

## Overview

Environment variables are the standard way to configure containerized applications. In Kubernetes, they can come from direct values, ConfigMaps, or Secrets. Portainer's application form makes it easy to configure all three methods.

## Method 1: Direct Value Environment Variables

When deploying an application in Portainer, scroll to the **Environment variables** section and add key-value pairs:

```yaml
# Equivalent YAML generated by Portainer
env:
  - name: APP_ENV
    value: "production"
  - name: LOG_LEVEL
    value: "info"
  - name: MAX_CONNECTIONS
    value: "100"
```

## Method 2: From ConfigMap

Reference keys from an existing ConfigMap:

```yaml
# Load specific keys from a ConfigMap
env:
  - name: DATABASE_URL
    valueFrom:
      configMapKeyRef:
        name: app-config
        key: database_url

# Load ALL keys from a ConfigMap as environment variables
envFrom:
  - configMapRef:
      name: app-config
```

## Method 3: From Secret

Reference sensitive values from a Kubernetes Secret:

```yaml
# Load a specific key from a Secret
env:
  - name: DB_PASSWORD
    valueFrom:
      secretKeyRef:
        name: app-secrets
        key: db-password

# Load ALL keys from a Secret as environment variables
envFrom:
  - secretRef:
      name: app-secrets
```

## Creating ConfigMaps and Secrets First

Before referencing them, create the ConfigMap or Secret:

```bash
# Create a ConfigMap from literal values
kubectl create configmap app-config \
  --from-literal=database_url=postgres://db:5432/mydb \
  --from-literal=redis_url=redis://cache:6379 \
  --namespace=production

# Create a Secret for sensitive values
kubectl create secret generic app-secrets \
  --from-literal=db-password=supersecret \
  --from-literal=api-key=myapikey \
  --namespace=production
```

## Setting Environment Variables in Portainer

In the Portainer application form:

1. Go to **Applications > Add application**.
2. Scroll to **Environment variables**.
3. Choose the type:
   - **Simple value**: Enter key and value directly.
   - **From ConfigMap**: Select the ConfigMap and key.
   - **From Secret**: Select the Secret and key.
4. Click **Add environment variable** for each one.

## Combining Multiple Sources

```yaml
# A realistic pod spec combining multiple env var sources
spec:
  containers:
    - name: app
      image: myapp:latest
      # Individual direct value
      env:
        - name: APP_VERSION
          value: "2.0.0"
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: app-secrets
              key: db-password
      # Load all non-sensitive config as env vars
      envFrom:
        - configMapRef:
            name: app-config
```

## Conclusion

Portainer simplifies environment variable management for Kubernetes apps by providing a form interface for all three injection methods. Always use Secrets for sensitive values rather than direct string values, and use ConfigMaps for non-sensitive configuration.
