Description: Deep-dive into each ICMPv6 Destination Unreachable code, when to generate each, how applications interpret them, and the correct firewall and router behaviors for each code.

## Introduction

The eight codes of ICMPv6 Destination Unreachable (Type 1) each represent a distinct failure scenario. Routers and endpoints generate different codes based on why the packet cannot be delivered. Applications receive these codes as socket errors. Firewalls must generate the correct code when rejecting traffic. Using the wrong code misleads diagnostics and can reveal or hide information about network topology.

## Code-by-Code Analysis

```text
Code 0: No Route to Destination
  Generated by: Router (not the destination)
  When: No routing table entry matches the destination address
  Application sees: ENETUNREACH (Network unreachable)
  Example: Packet to 2001:db8::1 when no route to 2001:db8::/32 exists

Code 1: Administratively Prohibited
  Generated by: Router or endpoint (firewall)
  When: Packet matches a "reject" firewall rule
  Application sees: ECONNREFUSED or EACCES (connection refused/access denied)
  Note: Tells the source traffic is blocked; use if transparency is acceptable
  Example: ip6tables -j REJECT --reject-with icmp6-adm-prohibited

Code 2: Beyond Scope of Source Address
  Generated by: Router or endpoint
  When: Source address scope is insufficient for the destination
  Example: Host uses link-local source address to reach global destination
           through a router (router rejects: link-local src cannot be routed)

Code 3: Address Unreachable
  Generated by: Router (after NDP failure)
  When: Neighbor Discovery fails to resolve the destination MAC address
  Application sees: EHOSTUNREACH (Host unreachable)
  Example: IPv6 address exists in routing table, but the host doesn't respond
           to Neighbor Solicitation on the link

Code 4: Port Unreachable
  Generated by: Destination host (not a router)
  When: No process is listening on the destination port (UDP only)
  Application sees: ECONNREFUSED (connection refused)
  Note: TCP sends RST instead of using ICMPv6 for port unreachable
  Example: UDP packet to 2001:db8::1:53 when no DNS server runs

Code 5: Source Address Failed Ingress/Egress Policy
  Generated by: Router performing uRPF or ingress filtering
  When: Packet fails Unicast Reverse Path Forwarding check
  Application sees: EACCES
  Example: Packet arrives on wrong interface relative to routing table
           (BCP 38 anti-spoofing filter)

Code 6: Reject Route
  Generated by: Router
  When: Routing table has an explicit "blackhole" or "reject" route entry
  Application sees: ENETUNREACH
  Example: ip -6 route add blackhole 2001:db8::/32

Code 7: Error in Source Routing Header
  Generated by: Router processing IPv6 Routing Header
  When: Segment Routing Header contains an error
  Example: Routing header with next-segment address that is unreachable
```

## Application Error Mapping

```python
import socket
import errno

# Map ICMPv6 Destination Unreachable codes to Python socket errors

CODE_TO_ERRNO = {
    0: (errno.ENETUNREACH,  "ENETUNREACH: Network unreachable"),
    1: (errno.ECONNREFUSED, "ECONNREFUSED: Connection refused (admin prohibited)"),
    2: (errno.EACCES,       "EACCES: Beyond scope of source address"),
    3: (errno.EHOSTUNREACH, "EHOSTUNREACH: Host unreachable"),
    4: (errno.ECONNREFUSED, "ECONNREFUSED: Port unreachable"),
    5: (errno.EACCES,       "EACCES: Source address policy failure"),
    6: (errno.ENETUNREACH,  "ENETUNREACH: Reject route"),
    7: (errno.EHOSTUNREACH, "EHOSTUNREACH: Source routing header error"),
}

def connect_with_error_diagnosis(host: str, port: int) -> str:
    """Attempt TCP connect and interpret ICMPv6 errors."""
    try:
        s = socket.socket(socket.AF_INET6, socket.SOCK_STREAM)
        s.settimeout(5)
        s.connect((host, port))
        s.close()
        return "Connected successfully"
    except OSError as e:
        error_map = {
            errno.ENETUNREACH:   "No route to host or reject route (Code 0 or 6)",
            errno.EHOSTUNREACH:  "Address unreachable / NDP failed (Code 3)",
            errno.ECONNREFUSED:  "Port closed or admin blocked (Code 4 or 1)",
            errno.EACCES:        "Policy rejection (Code 2 or 5)",
            errno.ETIMEDOUT:     "Timeout (possible PMTU black hole or packet loss)",
        }
        return error_map.get(e.errno, f"Unknown error: {e}")

# Test
print(connect_with_error_diagnosis("2001:db8::1", 80))
```

## Generating the Correct Code in ip6tables

```bash
# Code 0: No route (use when dropping due to missing route)
# This is automatically generated by the kernel for routing failures

# Code 1: Administratively prohibited (explicit reject)
sudo ip6tables -A FORWARD -d 2001:db8:bad::/48 \
    -j REJECT --reject-with icmp6-adm-prohibited

# Code 3: Address unreachable
sudo ip6tables -A FORWARD \
    -j REJECT --reject-with icmp6-addr-unreach

# Code 4: Port unreachable (for UDP port blocking)
sudo ip6tables -A INPUT -p udp --dport 123 \
    -j REJECT --reject-with icmp6-port-unreach

# Using DROP instead of REJECT (sends no ICMPv6 at all)
# Use when you want to hide that traffic is being filtered
sudo ip6tables -A FORWARD -s 2001:db8:suspect::/48 -j DROP

# List reject rules showing the reject-with type
sudo ip6tables -L -v -n | grep REJECT
```

## Conclusion

The correct use of ICMPv6 Destination Unreachable codes provides accurate feedback to sources about why their packets failed. Code 0 for routing failures, Code 1 for administrative policy blocks, Code 3 for NDP resolution failures, and Code 4 for closed UDP ports are the most common. Using REJECT with the appropriate `--reject-with` value in ip6tables generates the correct code for each scenario. When security requires hiding the reason for packet drops, use DROP instead to send no ICMPv6 error at all.
