# How to Configure K3s to Use containerd

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: K3s, containerd, Container Runtime, Kubernetes, SUSE Rancher, CRI, Registry

Description: Learn how to configure K3s's embedded containerd runtime, including registry mirrors, snapshot settings, custom configuration overlays, and runtime class configuration.

---

K3s ships with a built-in containerd runtime. Unlike Docker-based runtimes, containerd integrates directly with Kubernetes via the CRI without an additional shim layer. This guide covers how to configure and customize the embedded containerd.

---

## K3s containerd Configuration Locations

| File | Purpose |
|---|---|
| `/var/lib/rancher/k3s/agent/etc/containerd/config.toml` | Auto-generated by K3s — do not edit directly |
| `/var/lib/rancher/k3s/agent/etc/containerd/config.toml.tmpl` | Override template — edit this file |
| `/etc/rancher/k3s/registries.yaml` | Registry mirrors and credentials |

---

## Step 1: Configure Registry Mirrors

The simplest way to redirect image pulls to a mirror:

```yaml
# /etc/rancher/k3s/registries.yaml
mirrors:
  "docker.io":
    endpoint:
      - "https://mirror.gcr.io"
      - "https://registry.example.com"
  "registry.k8s.io":
    endpoint:
      - "https://k8s-mirror.example.com"

configs:
  "registry.example.com":
    auth:
      username: myuser
      password: mypassword
    tls:
      # Skip TLS verification for internal registries (dev only)
      insecure_skip_verify: false
      ca_file: /etc/ssl/certs/internal-ca.crt
```

Restart K3s to apply:

```bash
systemctl restart k3s
```

## Step 2: Customize containerd via Template

If you need to modify containerd settings beyond what K3s exposes, create a config template:

```toml
# /var/lib/rancher/k3s/agent/etc/containerd/config.toml.tmpl
version = 2

[plugins."io.containerd.internal.v1.opt"]
  path = "/var/lib/rancher/k3s/agent/containerd"

[plugins."io.containerd.grpc.v1.cri"]
  stream_server_address = "127.0.0.1"
  stream_server_port = "10010"
  enable_selinux = false
  enable_unprivileged_ports = true
  enable_unprivileged_icmp = true

  [plugins."io.containerd.grpc.v1.cri".containerd]
    # Use overlayfs snapshotter (default, most performant)
    snapshotter = "overlayfs"
    default_runtime_name = "runc"

    [plugins."io.containerd.grpc.v1.cri".containerd.runtimes.runc]
      runtime_type = "io.containerd.runc.v2"

      [plugins."io.containerd.grpc.v1.cri".containerd.runtimes.runc.options]
        SystemdCgroup = true
```

---

## Step 3: Add a GPU Runtime Class (Optional)

For nodes with NVIDIA GPUs, add the nvidia runtime to the template:

```toml
# Append to config.toml.tmpl
[plugins."io.containerd.grpc.v1.cri".containerd.runtimes.nvidia]
  runtime_type = "io.containerd.runc.v2"

  [plugins."io.containerd.grpc.v1.cri".containerd.runtimes.nvidia.options]
    BinaryName = "/usr/bin/nvidia-container-runtime"
```

Create the corresponding RuntimeClass in Kubernetes:

```yaml
apiVersion: node.k8s.io/v1
kind: RuntimeClass
metadata:
  name: nvidia
handler: nvidia
```

## Step 4: Verify containerd Configuration

```bash
# Check that K3s is using containerd
sudo k3s crictl info | jq .config.containerdEndpoint

# List running containers via containerd
sudo k3s crictl ps

# Check pulled images
sudo k3s crictl images

# Pull a test image to verify registry mirror
sudo k3s crictl pull nginx:latest
```

---

## Best Practices

- Use the `config.toml.tmpl` template instead of editing the auto-generated file — K3s overwrites it on restart.
- Enable `SystemdCgroup = true` for proper cgroup v2 support on modern Linux distributions.
- Use the `registries.yaml` for mirror configuration — it is the cleanest way to handle registry redirects in K3s.
