## Introduction

DHCPv6 assigns IPv6 addresses to virtual machines through a stateful protocol, unlike SLAAC which is stateless. In virtualization environments, DHCPv6 can run on the hypervisor host, within the virtual network (libvirt, OpenStack Neutron, NSX-T), or on a dedicated VM. DHCPv6 provides IPAM visibility — each lease is recorded with a timestamp and DUID — making it preferable to SLAAC for environments that require IP tracking.

## Kea DHCPv6 on the Hypervisor Host

```json
// /etc/kea/kea-dhcp6.conf
{
  "Dhcp6": {
    "interfaces-config": {
      // Listen on the bridge interface for VM traffic
      "interfaces": ["virbr0", "br0"]
    },
    "subnet6": [
      {
        "subnet": "2001:db8:vms::/64",
        "pools": [
          {
            "pool": "2001:db8:vms::100-2001:db8:vms::200"
          }
        ],
        "option-data": [
          {
            "name": "dns-servers",
            "data": "2001:db8::53, 2001:4860:4860::8888"
          },
          {
            "name": "domain-search",
            "data": "example.com"
          }
        ],
        "reservations": [
          {
            "hw-address": "52:54:00:ab:cd:01",
            "ip-addresses": ["2001:db8:vms::10"],
            "hostname": "webserver"
          },
          {
            "hw-address": "52:54:00:ab:cd:02",
            "ip-addresses": ["2001:db8:vms::11"],
            "hostname": "dbserver"
          }
        ]
      }
    ],
    "valid-lifetime": 3600,
    "renew-timer": 1800,
    "rebind-timer": 2700,
    "lease-database": {
      "type": "memfile",
      "name": "/var/lib/kea/dhcp6.leases"
    }
  }
}
```

```bash
# Enable and start Kea DHCPv6
systemctl enable --now kea-dhcp6-server

# Verify DHCPv6 leases
cat /var/lib/kea/dhcp6.leases

# Check leases via Kea API
curl -X POST "http://localhost:8080/" \
    -H "Content-Type: application/json" \
    -d '{"command": "lease6-get-all", "service": ["dhcp6"]}'
```

## dnsmasq DHCPv6 for KVM (libvirt)

```bash
# libvirt uses dnsmasq for DHCPv6 in virtual networks
# Configure via virsh or XML

# Check dnsmasq configuration generated by libvirt
cat /var/lib/libvirt/dnsmasq/default.conf | grep dhcp
cat /var/lib/libvirt/dnsmasq/default.hostsfile

# Add DHCPv6 host entry for a VM
virsh net-update default add ip-dhcp-host \
    '<host id="00:01:00:01:xx:xx:xx:xx:52:54:00:ab:cd:01" name="webserver" ip="fd00:vm:net::10"/>' \
    --live --config

# The DUID format: 00:01:00:01 = DUID-LLT, followed by time and MAC
# Simplified: use MAC-based DUID
virsh net-update default add ip-dhcp-host \
    '<host mac="52:54:00:ab:cd:01" name="webserver6" ip="fd00:vm:net::10"/>' \
    --live --config

# List current DHCPv6 leases
virsh net-dhcp-leases default | grep -i "ipv6\|fd00\|2001"
```

## OpenStack Neutron DHCPv6

```bash
# OpenStack Neutron runs dnsmasq for DHCPv6 on tenant networks

# Create subnet with DHCPv6 stateful
openstack subnet create \
    --network my-network \
    --ip-version 6 \
    --subnet-range 2001:db8:cloud::/64 \
    --dhcp \
    --ipv6-address-mode dhcpv6-stateful \
    --ipv6-ra-mode dhcpv6-stateful \
    --allocation-pool start=2001:db8:cloud::100,end=2001:db8:cloud::200 \
    my-ipv6-subnet

# Check DHCP agent is running
openstack network agent list | grep dhcp

# View DHCPv6 leases via Neutron
openstack port list --network my-network --long | grep "ip_address"
# Each port shows its IPv6 address from DHCPv6
```

## Proxmox Container DHCPv6

```bash
# LXC containers in Proxmox can use DHCPv6 from the host

# /etc/pve/lxc/100.conf — Container network config
net0: name=eth0,bridge=vmbr0,hwaddr=52:54:00:aa:bb:cc,ip=dhcp,ip6=dhcp

# Start container — it will request both DHCPv4 and DHCPv6
pct start 100

# Check container got IPv6 from DHCPv6
pct exec 100 -- ip -6 addr show
# Expected: inet6 2001:db8:vms::xx/64 scope global dynamic

# Check DHCPv6 lease on host
grep -i "2001:db8:vms" /var/lib/kea/dhcp6.leases
```

## Monitor DHCPv6 Leases

```python
#!/usr/bin/env python3
# monitor_dhcpv6_leases.py

import json
import requests
from datetime import datetime

KEA_API = "http://localhost:8080/"

def get_dhcpv6_leases() -> list:
    """Retrieve all DHCPv6 leases from Kea."""
    response = requests.post(
        KEA_API,
        json={"command": "lease6-get-all", "service": ["dhcp6"]}
    )
    result = response.json()
    if result[0]["result"] == 0:
        return result[0].get("arguments", {}).get("leases", [])
    return []

def print_lease_summary():
    leases = get_dhcpv6_leases()
    print(f"DHCPv6 Leases: {len(leases)}")
    for lease in leases:
        expire = datetime.fromtimestamp(lease.get("expire", 0))
        print(f"  {lease['ip-address']:40s}  {lease.get('hostname', 'unknown'):20s}  expires: {expire}")

print_lease_summary()
```

## Conclusion

DHCPv6 in virtualization environments provides centralized, trackable IPv6 address assignment for VMs and containers. Kea DHCPv6 running on the hypervisor host listens on bridge interfaces (virbr0, br0) and serves addresses to VMs via the `subnet6` configuration. libvirt uses dnsmasq for DHCPv6 in virtual networks, configurable via `virsh net-update`. OpenStack Neutron automatically deploys dnsmasq for DHCPv6 when subnets use `dhcpv6-stateful` mode. DHCPv6 reservations by MAC address (or DUID) ensure specific VMs always receive the same IPv6 address, enabling stable DNS records and firewall rules. Monitor leases via the Kea REST API or `virsh net-dhcp-leases` for IPAM tracking.
