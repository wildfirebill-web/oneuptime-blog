# How to Configure OSPF Totally Stubby Areas

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: OSPF, Totally Stubby, Cisco IOS, Routing, Optimization

Description: Learn how to configure OSPF totally stubby areas to block both external LSAs and inter-area summary LSAs, leaving only a default route for all non-local destinations.

## Totally Stubby vs. Stub Areas

A regular stub area blocks Type-4 and Type-5 external LSAs but still floods Type-3 inter-area summary LSAs (routes from other OSPF areas). A **totally stubby area** blocks Type-3, Type-4, and Type-5 LSAs—only a single default route from the ABR remains. This is the most aggressive routing table reduction possible in OSPF.

## LSA Types in a Totally Stubby Area

| LSA Type | Description | Allowed? |
|---|---|---|
| Type 1 | Router LSA | Yes |
| Type 2 | Network LSA | Yes |
| Type 3 | Summary (Inter-area) | No (except the default) |
| Type 4 | ASBR Summary | No |
| Type 5 | External Routes | No |

The only Type-3 LSA allowed is the 0.0.0.0/0 default route generated by the ABR.

## When to Use Totally Stubby Areas

Totally stubby areas are ideal for:
- Branch sites with a single connection to the backbone
- Sites where all external traffic must go through the ABR anyway
- Reducing SPF computation overhead on small branch routers with limited CPU/RAM

## Step 1: Configure the ABR for Totally Stubby Area

The `no-summary` keyword is the difference between stub and totally stubby—it's a **Cisco-specific extension** applied only on the ABR:

```
! On the ABR - configure Area 1 as totally stubby
ABR(config)# router ospf 1
ABR(config-router)# area 1 stub no-summary

! The ABR automatically injects a default route into Area 1
! All Type-3, Type-4, and Type-5 LSAs are blocked from Area 1
```

## Step 2: Configure Internal Routers (Stub Only—Not Totally Stubby)

Internal routers in the area only need to know they're in a stub area. The `no-summary` keyword is **not configured on internal routers**—it's ABR-only:

```
! On internal router R1 in Area 1
R1(config)# router ospf 1
R1(config-router)# router-id 172.16.1.1
R1(config-router)# network 172.16.1.0 0.0.0.255 area 1
! Configure as stub (NOT no-summary - that's only on ABR)
R1(config-router)# area 1 stub
```

## Step 3: Verify the Totally Stubby Configuration

```
! On the ABR - check area configuration
ABR# show ip ospf | begin Area 1

Area 1
  Number of interfaces in this area is 2
  It is a stub area, no summary LSA in this area
  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
  This message confirms totally stubby configuration

! On internal router R1 - verify minimal routing table
R1# show ip route ospf

! Should only see:
! O IA  0.0.0.0/0 [110/11] via 172.16.1.254     <- Only the default route
! O     172.16.2.0/24 [110/2] via 172.16.1.254  <- Intra-area routes

! No inter-area summaries, no external routes
```

## Step 4: Set Default Route Cost

Control which ABR is preferred for the default route when multiple ABRs serve the area:

```
! Primary ABR - low cost default route
ABR1(config-router)# area 1 stub no-summary
ABR1(config-router)# area 1 default-cost 5

! Backup ABR - high cost default route
ABR2(config-router)# area 1 stub no-summary
ABR2(config-router)# area 1 default-cost 50
```

Internal routers prefer ABR1 (cost 5) for all traffic, failing over to ABR2 (cost 50) if ABR1 is unavailable.

## Step 5: Compare with Stub Area

Run this comparison after configuring:

```
! Show routing table size (should be much smaller with totally stubby)
R1# show ip route summary

! Totally stubby (expected): 1 default + few intra-area routes
! Regular stub (expected): many inter-area routes + 1 default
```

## Limitations

- OSPF virtual links cannot traverse stub or totally stubby areas
- ASBRs cannot be placed inside totally stubby areas (they redistribute external routes)
- If the branch needs specific inter-area routes (not just a default), use a regular stub area instead

## Conclusion

Totally stubby areas are the most streamlined OSPF area type, leaving only intra-area routes and one default route. Configure `area X stub no-summary` only on the ABR; internal routers use only `area X stub`. Use `area X default-cost` for load balancing or failover between multiple ABRs serving the same totally stubby area.
