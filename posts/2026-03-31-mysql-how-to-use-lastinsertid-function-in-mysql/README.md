# How to Use LAST_INSERT_ID() Function in MySQL

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: MySQL, LAST_INSERT_ID, Auto Increment, SQL Functions

Description: Learn how to use LAST_INSERT_ID() in MySQL to retrieve the auto-generated ID from the most recent INSERT operation in the current session.

---

## What is LAST_INSERT_ID()

MySQL's `LAST_INSERT_ID()` function returns the auto-generated value (the `AUTO_INCREMENT` value) that was inserted by the most recent `INSERT` statement in the current session.

Key properties:
- Session-specific: only reflects inserts in the current connection
- Returns the first ID generated if multiple rows were inserted
- Returns 0 if no INSERT has generated an auto-increment value yet
- Not affected by inserts in other connections

Syntax:

```sql
LAST_INSERT_ID()
LAST_INSERT_ID(expr)  -- sets and returns expr
```

## Basic Usage

```sql
INSERT INTO employees (name, department) VALUES ('Alice Smith', 'Engineering');
SELECT LAST_INSERT_ID();
-- Output: the ID assigned to Alice's row
```

## Typical Use Case - Getting Inserted ID

```sql
-- Insert a parent record
INSERT INTO orders (user_id, total) VALUES (1, 150.00);

-- Get the new order ID
SET @order_id = LAST_INSERT_ID();

-- Insert child records using the parent ID
INSERT INTO order_items (order_id, product_id, quantity)
VALUES (@order_id, 42, 3), (@order_id, 17, 1);
```

## Multi-Row INSERT Returns First ID

When inserting multiple rows, `LAST_INSERT_ID()` returns only the first generated ID:

```sql
INSERT INTO products (name) VALUES ('Widget A'), ('Widget B'), ('Widget C');
SELECT LAST_INSERT_ID();
-- Returns the ID for 'Widget A' only (the first of the three inserts)
```

To get all three IDs:

```sql
SELECT id FROM products WHERE id >= LAST_INSERT_ID() LIMIT 3;
```

## LAST_INSERT_ID() in Stored Procedures

```sql
DELIMITER //
CREATE PROCEDURE create_order(
  IN p_user_id INT,
  IN p_total DECIMAL(10, 2),
  OUT p_order_id INT
)
BEGIN
  INSERT INTO orders (user_id, total) VALUES (p_user_id, p_total);
  SET p_order_id = LAST_INSERT_ID();
END //
DELIMITER ;

CALL create_order(1, 99.99, @new_order_id);
SELECT @new_order_id;
```

## LAST_INSERT_ID() with Transactions

`LAST_INSERT_ID()` is not affected by `ROLLBACK` - it retains the value even if the transaction is rolled back:

```sql
START TRANSACTION;
INSERT INTO temp_log (message) VALUES ('test');
SET @temp_id = LAST_INSERT_ID();
ROLLBACK;

SELECT @temp_id;  -- Still holds the ID even after rollback
```

## LAST_INSERT_ID(expr) - Manual Counter

The two-argument form sets a value for `LAST_INSERT_ID()` without performing an INSERT:

```sql
-- Create a simple sequence counter
CREATE TABLE counters (
  name VARCHAR(50) PRIMARY KEY,
  value INT NOT NULL DEFAULT 0
);

INSERT INTO counters VALUES ('order_seq', 0);

-- Atomically increment and retrieve
UPDATE counters
SET value = LAST_INSERT_ID(value + 1)
WHERE name = 'order_seq';

SELECT LAST_INSERT_ID();  -- Returns the new value
```

## Application Code Usage

In Python:

```python
cursor.execute(
    "INSERT INTO employees (name, department) VALUES (%s, %s)",
    ("Alice Smith", "Engineering")
)
new_id = cursor.lastrowid  # Equivalent to LAST_INSERT_ID()
print(f"New employee ID: {new_id}")
```

In Node.js (mysql2):

```javascript
const [result] = await conn.execute(
    'INSERT INTO employees (name, department) VALUES (?, ?)',
    ['Alice Smith', 'Engineering']
);
const newId = result.insertId;  // Equivalent to LAST_INSERT_ID()
console.log('New employee ID:', newId);
```

## Summary

`LAST_INSERT_ID()` retrieves the auto-increment value generated by the most recent INSERT in the current session. It is session-specific and safe for concurrent applications. Use it to chain parent-child INSERT statements, implement custom sequence counters with the `LAST_INSERT_ID(expr)` form, or return generated IDs from stored procedures. Most client libraries also expose this through `cursor.lastrowid` or `result.insertId`.
