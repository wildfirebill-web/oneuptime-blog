# How to Use Bitnami Redis Helm Chart in Kubernetes

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Redis, Helm, Kubernetes, Bitnami, High Availability, DevOps

Description: Deploy Redis on Kubernetes using the Bitnami Redis Helm chart with standalone and high-availability replication modes, custom configuration, and TLS support.

---

## Why Use the Bitnami Redis Helm Chart

The Bitnami Redis Helm chart is the most widely used Redis chart for Kubernetes. It supports:
- Standalone mode (single pod)
- Replication mode (primary + replicas with Sentinel)
- TLS encryption
- Persistent storage
- Prometheus metrics via redis-exporter

## Add the Bitnami Repository

```bash
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update
helm search repo bitnami/redis
```

## Install Standalone Redis

```bash
helm install my-redis bitnami/redis \
    --namespace redis \
    --create-namespace \
    --set architecture=standalone \
    --set auth.password=strongpassword123 \
    --set primary.persistence.size=8Gi
```

## Install Redis with Replication (Recommended for Production)

```bash
helm install my-redis bitnami/redis \
    --namespace redis \
    --create-namespace \
    --set architecture=replication \
    --set auth.password=strongpassword123 \
    --set replica.replicaCount=2 \
    --set primary.persistence.size=8Gi \
    --set replica.persistence.size=8Gi
```

## Custom values.yaml

Create `redis-values.yaml`:

```yaml
architecture: replication

auth:
  enabled: true
  password: "strongpassword123"

primary:
  resources:
    requests:
      cpu: 100m
      memory: 128Mi
    limits:
      cpu: 500m
      memory: 512Mi
  persistence:
    enabled: true
    size: 8Gi
    storageClass: standard

replica:
  replicaCount: 2
  resources:
    requests:
      cpu: 100m
      memory: 128Mi
    limits:
      cpu: 250m
      memory: 256Mi
  persistence:
    enabled: true
    size: 8Gi

sentinel:
  enabled: true
  masterSet: mymaster
  downAfterMilliseconds: 5000
  failoverTimeout: 10000

metrics:
  enabled: true
  serviceMonitor:
    enabled: true
    namespace: monitoring

commonConfiguration: |
  maxmemory 400mb
  maxmemory-policy allkeys-lru
  save 900 1
  save 300 10
```

Install with custom values:

```bash
helm install my-redis bitnami/redis \
    -f redis-values.yaml \
    --namespace redis \
    --create-namespace
```

## Verify the Deployment

```bash
kubectl get pods -n redis
kubectl get svc -n redis
kubectl get pvc -n redis

# Get the Redis password
kubectl get secret -n redis my-redis -o jsonpath="{.data.redis-password}" | base64 --decode
```

## Connect to Redis

```bash
# Port-forward to primary
kubectl port-forward -n redis svc/my-redis-master 6379:6379

# Connect
redis-cli -p 6379 -a strongpassword123 PING

# Connect via exec
kubectl exec -n redis my-redis-master-0 -- redis-cli -a strongpassword123 PING
```

## Upgrading the Chart

```bash
helm upgrade my-redis bitnami/redis \
    -f redis-values.yaml \
    --namespace redis
```

Check chart version:

```bash
helm list -n redis
helm history my-redis -n redis
```

## Enable TLS

```yaml
tls:
  enabled: true
  autoGenerated: true
  certificatesSecret: ""
  certFilename: "tls.crt"
  keyFilename: "tls.key"
  cacertFilename: "ca.crt"
```

## Using the Sentinel Service

When Sentinel is enabled, connect to the Sentinel service:

```python
import redis

sentinel = redis.Sentinel(
    [("my-redis.redis.svc.cluster.local", 26379)],
    password="strongpassword123"
)

master = sentinel.master_for("mymaster")
master.set("key", "value")
```

## Uninstalling

```bash
helm uninstall my-redis -n redis
kubectl delete pvc -n redis -l app.kubernetes.io/name=redis
```

## Summary

The Bitnami Redis Helm chart deploys production-ready Redis on Kubernetes in standalone or replication-with-Sentinel mode. A custom `values.yaml` file controls resource limits, persistent storage, Sentinel configuration, and Prometheus metrics. Sentinel-aware clients connect to the Sentinel service endpoint for automatic failover without needing to know the primary's IP address.
