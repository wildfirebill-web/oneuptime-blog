# How to Analyze Fleet Management Data in ClickHouse

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: ClickHouse, Fleet Management, Logistics, Telematics, Supply Chain

Description: Analyze vehicle telematics, fuel consumption, route adherence, and driver behavior in ClickHouse to reduce fleet operating costs and improve delivery performance.

---

Fleet managers overseeing hundreds or thousands of vehicles need real-time visibility into vehicle location, fuel consumption, driver behavior, and maintenance status. ClickHouse handles the high-frequency telematics data streams generated by GPS and OBD devices and provides fast analytical queries for operational dashboards.

## Vehicle Telematics Table

```sql
CREATE TABLE vehicle_telematics (
    ping_id         UUID,
    vehicle_id      UInt32,
    driver_id       UInt32,
    fleet_id        UInt16,
    recorded_at     DateTime,
    lat             Float64,
    lon             Float64,
    speed_kmh       Float32,
    heading         UInt16,
    engine_on       UInt8,
    fuel_level_pct  Float32,
    odometer_km     UInt64,
    harsh_brake     UInt8,
    harsh_accel     UInt8,
    idling          UInt8,
    engine_rpm      UInt16,
    coolant_temp_c  Float32
) ENGINE = MergeTree()
ORDER BY (fleet_id, vehicle_id, recorded_at)
PARTITION BY toYYYYMM(recorded_at)
TTL recorded_at + INTERVAL 2 YEAR;
```

## Fuel Consumption by Vehicle

```sql
SELECT
    vehicle_id,
    round(max(odometer_km) - min(odometer_km), 0) AS km_driven,
    round(avg(fuel_level_pct), 1)                 AS avg_fuel_pct,
    countIf(idling = 1)                           AS idle_pings
FROM vehicle_telematics
WHERE recorded_at >= today() - 7
GROUP BY vehicle_id
ORDER BY idle_pings DESC
LIMIT 20;
```

## Driver Behavior Scoring

```sql
SELECT
    driver_id,
    count()                          AS pings,
    sum(harsh_brake)                 AS harsh_brakes,
    sum(harsh_accel)                 AS harsh_accels,
    countIf(speed_kmh > 120)         AS speeding_events,
    round(
        100 - (sum(harsh_brake) + sum(harsh_accel) + countIf(speed_kmh > 120)) * 1.0 / count() * 100,
        1
    ) AS safety_score
FROM vehicle_telematics
WHERE recorded_at >= today() - 30
GROUP BY driver_id
HAVING pings > 100
ORDER BY safety_score ASC
LIMIT 20;
```

## Fleet Idle Time Analysis

```sql
SELECT
    vehicle_id,
    toDate(recorded_at)   AS day,
    sum(idling)           AS idle_pings,
    count()               AS total_pings,
    round(100.0 * sum(idling) / count(), 1) AS idle_pct
FROM vehicle_telematics
WHERE recorded_at >= today() - 14
  AND engine_on = 1
GROUP BY vehicle_id, day
HAVING idle_pct > 30
ORDER BY day, idle_pct DESC;
```

## Speeding Events Heatmap by Hour

```sql
SELECT
    toHour(recorded_at)         AS hour,
    toDayOfWeek(recorded_at)    AS weekday,
    count()                     AS speeding_events
FROM vehicle_telematics
WHERE speed_kmh > 120
  AND recorded_at >= today() - 30
GROUP BY hour, weekday
ORDER BY weekday, hour;
```

## Vehicle Health - High Coolant Temperature Alerts

```sql
SELECT
    vehicle_id,
    max(coolant_temp_c)  AS max_temp,
    count()              AS high_temp_events
FROM vehicle_telematics
WHERE coolant_temp_c > 105   -- threshold for alert
  AND recorded_at >= now() - INTERVAL 24 HOUR
GROUP BY vehicle_id
HAVING high_temp_events > 5
ORDER BY max_temp DESC;
```

## Distance Driven per Driver per Day

```sql
SELECT
    driver_id,
    toDate(recorded_at)                        AS day,
    max(odometer_km) - min(odometer_km)        AS km_driven
FROM vehicle_telematics
WHERE recorded_at >= today() - 7
GROUP BY driver_id, day
ORDER BY km_driven DESC
LIMIT 30;
```

## Summary

ClickHouse is an ideal backend for fleet telematics analytics due to its high ingestion rate and fast aggregation over time-series data. Driver behavior scoring, idle time analysis, and vehicle health monitoring all run in seconds over millions of ping records. TTL-based retention keeps storage costs manageable while preserving the historical data needed for trend analysis and compliance reporting.
