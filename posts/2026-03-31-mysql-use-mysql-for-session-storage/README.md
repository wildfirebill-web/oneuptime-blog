# How to Use MySQL for Session Storage

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: MySQL, Session, Authentication, Security, Performance

Description: Learn how to implement server-side session storage in MySQL with an optimized schema, expiry cleanup, and application integration for Node.js and Python web frameworks.

---

Storing sessions in MySQL centralizes session state across multiple application servers, enabling horizontal scaling without sticky sessions. While Redis is faster, MySQL session storage is simpler to operate when you already depend on MySQL and do not need sub-millisecond session lookup latency.

## Schema Design

Design the sessions table with a short-lived, indexed expiry column to enable efficient cleanup:

```sql
CREATE TABLE sessions (
  session_id   CHAR(128)    NOT NULL,
  user_id      BIGINT UNSIGNED,
  data         JSON,
  ip_address   VARCHAR(45),
  user_agent   TEXT,
  created_at   DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  last_active  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  expires_at   DATETIME     NOT NULL,
  PRIMARY KEY (session_id),
  INDEX idx_user_id (user_id),
  INDEX idx_expires_at (expires_at)
) ENGINE=InnoDB;
```

The `session_id` is a random 128-character token generated by the application. The `expires_at` index makes cleanup queries fast.

## Creating and Reading Sessions

```sql
-- Create a new session (30-minute expiry)
INSERT INTO sessions (session_id, user_id, data, ip_address, expires_at)
VALUES (
  'a3f8d2c1...',
  42,
  '{"cart_id": 99, "locale": "en-US"}',
  '203.0.113.10',
  NOW() + INTERVAL 30 MINUTE
);

-- Validate and read a session (atomically extend expiry)
UPDATE sessions
SET
  last_active = NOW(),
  expires_at  = NOW() + INTERVAL 30 MINUTE
WHERE session_id = 'a3f8d2c1...'
  AND expires_at > NOW();

SELECT user_id, data FROM sessions
WHERE session_id = 'a3f8d2c1...'
  AND expires_at > NOW();
```

## Deleting Sessions on Logout

```sql
DELETE FROM sessions WHERE session_id = 'a3f8d2c1...';
```

## Expiry Cleanup with MySQL Events

Schedule automatic cleanup with a MySQL Event Scheduler:

```sql
CREATE EVENT clean_expired_sessions
ON SCHEDULE EVERY 10 MINUTE
DO
  DELETE FROM sessions WHERE expires_at < NOW() LIMIT 5000;
```

The `LIMIT 5000` prevents a single large delete from blocking the table for too long. Enable the event scheduler if not already active:

```sql
SET GLOBAL event_scheduler = ON;
```

## Integration with Express.js (Node.js)

Use `express-mysql-session` to transparently back Express sessions with MySQL:

```javascript
const session = require('express-session');
const MySQLStore = require('express-mysql-session')(session);

const sessionStore = new MySQLStore({
  host: process.env.DB_HOST,
  user: process.env.DB_USER,
  password: process.env.DB_PASS,
  database: process.env.DB_NAME,
  expiration: 1800000,   // 30 minutes in ms
  clearExpired: true,
  checkExpirationInterval: 600000
});

app.use(session({
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  store: sessionStore,
  cookie: { secure: true, httpOnly: true, sameSite: 'strict' }
}));
```

## Summary

MySQL session storage works well for applications that already depend on MySQL and need multi-server session sharing without adding Redis. Use a random 128-character `session_id` as the primary key, index the `expires_at` column, extend expiry on every read to implement sliding expiry, and schedule a recurring cleanup event to purge expired rows. For applications with thousands of requests per second, migrate to Redis for lower latency.
