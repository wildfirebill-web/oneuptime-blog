# How to Set Up IPv6 Access Log Analysis with GoAccess

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: IPv6, GoAccess, Web Analytics, Nginx, Log Analysis

Description: Configure GoAccess to parse and visualize web server access logs containing IPv6 addresses, with real-time terminal dashboards and HTML report generation.

## Introduction

GoAccess is a real-time web log analyzer and viewer that works in a terminal or generates HTML reports. It supports IPv6 addresses in access logs without additional configuration, since it extracts the first field as the client IP. This guide covers log format configuration, running GoAccess on IPv6 logs, and generating reports.

## Step 1: Configure GoAccess Log Format

```apache
# /etc/goaccess/goaccess.conf

# For nginx combined log format

log-format COMBINED

# For custom nginx format with IPv6 in brackets
# log-format %h - %^ [%d:%t %^] "%r" %s %b
# date-format %d/%b/%Y
# time-format %H:%M:%S

# If nginx logs IPv6 as [addr]:port, use this custom format:
log-format %h - %^ [%d:%t %^] "%r" %s %b "%R" "%u"
date-format %d/%b/%Y
time-format %H:%M:%S
```

## Step 2: Run GoAccess on Nginx Logs

```bash
# Basic terminal report
goaccess /var/log/nginx/access.log -c

# Specify format explicitly (combined log)
goaccess /var/log/nginx/access.log \
    --log-format=COMBINED \
    -o /var/www/html/report.html

# Analyze multiple log files including rotated logs
zcat /var/log/nginx/access.log.*.gz | \
    cat - /var/log/nginx/access.log | \
    goaccess --log-format=COMBINED -o /tmp/full-report.html

# Filter only IPv6 requests before passing to GoAccess
grep -E '^[0-9a-fA-F:]{3,39} ' /var/log/nginx/access.log | \
    goaccess --log-format=COMBINED -o /tmp/ipv6-only-report.html
```

## Step 3: Real-Time Analysis

```bash
# Real-time terminal dashboard (refreshes as log grows)
tail -f /var/log/nginx/access.log | \
    goaccess --log-format=COMBINED --real-time-html \
    -o /var/www/html/realtime.html

# Or pipe from systemd journal
journalctl -u nginx -f --output=cat | \
    goaccess --log-format=COMBINED
```

## Step 4: Custom Log Format for IPv6

If your Nginx logs IPv6 in a non-standard way, define a custom format:

```nginx
# nginx.conf: log IPv6 addresses clearly
log_format ipv6_combined '$remote_addr - $remote_user [$time_local] '
                         '"$request" $status $body_bytes_sent '
                         '"$http_referer" "$http_user_agent" '
                         'ipv=$server_protocol';

access_log /var/log/nginx/access.log ipv6_combined;
```

```bash
# GoAccess config for this format
# %h = remote_addr, %^ = ignored fields, %d = date, %t = time
# %r = request, %s = status, %b = bytes, %R = referer, %u = user agent
goaccess /var/log/nginx/access.log \
    --log-format="%h - %^ [%d:%t %^] \"%r\" %s %b \"%R\" \"%u\" ipv=%^" \
    --date-format="%d/%b/%Y" \
    --time-format="%H:%M:%S" \
    -o /tmp/report.html
```

## Step 5: GoAccess HTML Report for IPv6 Stats

The HTML report generated by GoAccess includes:

- **Visitors** panel: lists unique IPv6 and IPv4 addresses with hit/bandwidth counts
- **Hosts** panel: top clients including full IPv6 addresses
- **Not Found** panel: 404s by IPv6 client
- **OS/Browser** panels: user agent breakdown

```bash
# Generate a full HTML report with GeoIP for IPv6
goaccess /var/log/nginx/access.log \
    --log-format=COMBINED \
    --geoip-database=/etc/geoip/GeoLite2-City.mmdb \
    -o /var/www/html/ipv6-geo-report.html

# The report includes country flags and city for IPv6 sources
# (requires MaxMind GeoLite2-City.mmdb with IPv6 support)
```

## Step 6: Automate Daily IPv6 Reports

```bash
#!/bin/bash
# /etc/cron.daily/goaccess-ipv6-report

REPORT_DIR="/var/www/html/reports"
LOG_FILE="/var/log/nginx/access.log"
DATE=$(date +%Y-%m-%d)

mkdir -p "$REPORT_DIR"

# Full report
goaccess "$LOG_FILE" \
    --log-format=COMBINED \
    --geoip-database=/etc/geoip/GeoLite2-City.mmdb \
    -o "${REPORT_DIR}/full-${DATE}.html" 2>/dev/null

# IPv6-only report
grep -E '^[0-9a-fA-F:]{3,39} ' "$LOG_FILE" | \
    goaccess --log-format=COMBINED \
    -o "${REPORT_DIR}/ipv6-only-${DATE}.html" 2>/dev/null

echo "Reports generated in ${REPORT_DIR}"
```

## Conclusion

GoAccess handles IPv6 addresses in standard web server log formats without special configuration since it treats the first log field as the client IP regardless of format (IPv4 or IPv6). Filter logs to IPv6-only using `grep` for colon-containing addresses before passing to GoAccess for IPv6-focused reports. Combine GoAccess with the MaxMind GeoLite2 database (which supports IPv6) for geographic reports of IPv6 traffic. Use `tail -f | goaccess` with `--real-time-html` for live dashboards.
