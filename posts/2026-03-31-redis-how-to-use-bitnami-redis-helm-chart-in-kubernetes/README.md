# How to Use Bitnami Redis Helm Chart in Kubernetes

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Redis, Kubernetes, Helm, Bitnami, High Availability

Description: Learn how to deploy Redis on Kubernetes using the Bitnami Redis Helm chart, including standalone, replication, and sentinel configurations with production-ready settings.

---

## Why Use the Bitnami Redis Helm Chart

The Bitnami Redis Helm chart is one of the most widely used Kubernetes Redis deployments. It supports:

- Standalone mode (single node)
- Master/slave replication with Sentinel
- Redis Cluster mode
- TLS encryption
- Authentication
- Persistent volumes
- Custom configuration

## Adding the Bitnami Repository

```bash
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update

# View available chart versions
helm search repo bitnami/redis --versions | head -10

# View chart documentation
helm show readme bitnami/redis
helm show values bitnami/redis > redis-default-values.yaml
```

## Installing a Standalone Redis

```bash
# Simple standalone installation
helm install redis bitnami/redis \
  --namespace redis \
  --create-namespace \
  --set auth.password=your-strong-password \
  --set architecture=standalone

# Check pods
kubectl -n redis get pods
kubectl -n redis get svc
```

## Installing Redis with Replication and Sentinel

```yaml
# redis-values.yaml
architecture: replication

auth:
  enabled: true
  password: "your-strong-redis-password"

master:
  count: 1
  resources:
    requests:
      memory: 256Mi
      cpu: 100m
    limits:
      memory: 1Gi
      cpu: 500m
  persistence:
    enabled: true
    size: 8Gi
    storageClass: standard

replica:
  replicaCount: 2
  resources:
    requests:
      memory: 256Mi
      cpu: 100m
    limits:
      memory: 1Gi
      cpu: 500m
  persistence:
    enabled: true
    size: 8Gi
    storageClass: standard

sentinel:
  enabled: true
  masterSet: mymaster
  quorum: 2
  resources:
    requests:
      memory: 64Mi
      cpu: 50m
    limits:
      memory: 128Mi
      cpu: 100m

metrics:
  enabled: true
  serviceMonitor:
    enabled: false  # Set to true if you have Prometheus Operator

commonConfiguration: |
  maxmemory 800mb
  maxmemory-policy allkeys-lru
  appendonly yes
  appendfsync everysec
```

```bash
helm install redis bitnami/redis \
  --namespace redis \
  --create-namespace \
  -f redis-values.yaml
```

## Installing Redis Cluster

```yaml
# redis-cluster-values.yaml
cluster:
  nodes: 6
  replicas: 1

auth:
  enabled: true
  password: "your-strong-redis-password"

persistence:
  enabled: true
  size: 8Gi
  storageClass: standard

resources:
  requests:
    memory: 256Mi
    cpu: 100m
  limits:
    memory: 1Gi
    cpu: 500m
```

```bash
helm install redis-cluster bitnami/redis-cluster \
  --namespace redis \
  --create-namespace \
  -f redis-cluster-values.yaml
```

## Verifying the Deployment

```bash
# Check pods and services
kubectl -n redis get all

# Get the Redis password
export REDIS_PASSWORD=$(kubectl get secret --namespace redis redis -o jsonpath="{.data.redis-password}" | base64 -d)
echo $REDIS_PASSWORD

# Connect via redis-cli
kubectl -n redis run redis-client --rm --tty -i \
  --restart='Never' \
  --image docker.io/bitnami/redis:7.2 \
  --env REDIS_PASSWORD=$REDIS_PASSWORD \
  --command -- \
  redis-cli -h redis-master -a $REDIS_PASSWORD
```

## Connecting from Applications

For the replication+sentinel setup, get the connection details:

```bash
# Master service
kubectl -n redis get svc redis-master
# redis-master.redis.svc.cluster.local:6379

# Sentinel service
kubectl -n redis get svc redis
# redis.redis.svc.cluster.local:26379 (sentinel port)
```

Example Python connection:

```python
import redis.sentinel

sentinel = redis.sentinel.Sentinel(
    [('redis.redis.svc.cluster.local', 26379)],
    sentinel_kwargs={'password': 'your-strong-redis-password'},
    password='your-strong-redis-password'
)

master = sentinel.master_for('mymaster', socket_timeout=0.5)
master.set('key', 'value')
print(master.get('key'))
```

## Upgrading the Chart

```bash
# Upgrade to a newer chart version
helm upgrade redis bitnami/redis \
  --namespace redis \
  -f redis-values.yaml

# Upgrade with a specific version
helm upgrade redis bitnami/redis \
  --namespace redis \
  --version 19.0.0 \
  -f redis-values.yaml

# Check upgrade history
helm history redis -n redis
```

## Rollback

```bash
# Roll back to previous release
helm rollback redis 1 -n redis

# Check rollback status
helm status redis -n redis
```

## Enabling TLS

```yaml
# Add to redis-values.yaml for TLS
tls:
  enabled: true
  autoGenerated: true
  existingSecret: ""
  certFilename: tls.crt
  certKeyFilename: tls.key
  certCAFilename: ca.crt
```

## Enabling Prometheus Metrics

```yaml
# Add to redis-values.yaml
metrics:
  enabled: true
  image:
    registry: docker.io
    repository: bitnami/redis-exporter
    tag: latest
  serviceMonitor:
    enabled: true
    namespace: monitoring
    interval: 30s
    scrapeTimeout: 10s
  prometheusRule:
    enabled: true
    rules:
      - alert: RedisDown
        expr: redis_up == 0
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "Redis instance is down"
```

## Uninstalling

```bash
# Uninstall the release
helm uninstall redis -n redis

# Note: PVCs are NOT deleted automatically
kubectl -n redis get pvc
kubectl -n redis delete pvc -l app.kubernetes.io/instance=redis
```

## Summary

The Bitnami Redis Helm chart provides a production-ready Redis deployment with support for standalone, replication with Sentinel, and Redis Cluster modes. Configure authentication, persistence, and resource limits in a values YAML file, then deploy with helm install. Use Sentinel mode for automatic primary failover in replication setups, and retrieve the auto-generated password from the Kubernetes secret. Enable Prometheus metrics with serviceMonitor for integrated monitoring, and use helm upgrade with the same values file for safe updates.
