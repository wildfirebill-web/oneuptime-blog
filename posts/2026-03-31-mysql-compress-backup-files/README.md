# How to Compress MySQL Backup Files

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: MySQL, Backup, Compression, Storage, Gzip

Description: Learn how to compress MySQL backup files using gzip, bzip2, and zstd to reduce storage costs and backup file sizes by up to 90%.

---

MySQL backup files generated by `mysqldump` are plain text SQL, which compresses extremely well - often achieving 80-90% size reduction. Compressing backups is essential for managing storage costs, speeding up file transfers to remote backup destinations, and fitting more backup history on limited disk space.

## Why Compress Backups?

A typical compression comparison for a 10 GB MySQL database:

```text
Uncompressed .sql  - 10 GB
gzip .sql.gz       - 1.2 GB  (88% reduction)
bzip2 .sql.bz2     - 0.9 GB  (91% reduction)
zstd .sql.zst      - 1.1 GB  (89% reduction, much faster)
```

## Compressing During Backup (Inline Pipe)

The most efficient approach is compressing output as it is generated:

```bash
# With gzip
mysqldump -u root -p \
  --single-transaction \
  myapp \
  | gzip > /backup/myapp_$(date +%Y%m%d).sql.gz

# With bzip2
mysqldump -u root -p \
  --single-transaction \
  myapp \
  | bzip2 > /backup/myapp_$(date +%Y%m%d).sql.bz2

# With zstd (fast, good ratio)
mysqldump -u root -p \
  --single-transaction \
  myapp \
  | zstd -o /backup/myapp_$(date +%Y%m%d).sql.zst
```

## Compressing Existing Backup Files

```bash
# Compress in place (removes original)
gzip /backup/myapp_old.sql

# Compress keeping original
gzip -k /backup/myapp_old.sql

# Compress with maximum compression
gzip -9 /backup/myapp_old.sql
```

## Parallel Compression with pigz

`pigz` uses multiple CPU cores for faster gzip compression:

```bash
apt install pigz

mysqldump -u root -p myapp \
  | pigz -p 4 > /backup/myapp_$(date +%Y%m%d).sql.gz
```

## Adjusting Compression Level

Higher levels mean smaller files but slower compression:

```bash
# Fast compression (level 1)
mysqldump -u root -p myapp | gzip -1 > /backup/myapp_fast.sql.gz

# Best compression (level 9)
mysqldump -u root -p myapp | gzip -9 > /backup/myapp_best.sql.gz

# Default (level 6) is a good balance
mysqldump -u root -p myapp | gzip > /backup/myapp.sql.gz
```

## Verifying Compressed Backup Integrity

Always verify compressed files are not corrupted after creation:

```bash
# Test gzip integrity
gzip -t /backup/myapp_20260331.sql.gz && echo "File OK"

# Test bzip2 integrity
bzip2 -t /backup/myapp_20260331.sql.bz2 && echo "File OK"

# Check size and dates
ls -lh /backup/*.sql.gz
```

## Previewing Compressed Backup Content

View the backup without fully decompressing:

```bash
# Show first 30 lines
zcat /backup/myapp_20260331.sql.gz | head -30

# Search for a specific table
zcat /backup/myapp_20260331.sql.gz | grep "CREATE TABLE"
```

## Restoring from Compressed Backups

```bash
# From .gz
gunzip < /backup/myapp_20260331.sql.gz | mysql -u root -p myapp

# Using zcat shortcut
zcat /backup/myapp_20260331.sql.gz | mysql -u root -p myapp

# From .bz2
bzcat /backup/myapp_20260331.sql.bz2 | mysql -u root -p myapp

# From .zst
zstd -d < /backup/myapp_20260331.sql.zst | mysql -u root -p myapp
```

## Encrypting Compressed Backups

Combine compression with encryption for secure offsite storage:

```bash
mysqldump -u root -p myapp \
  | gzip \
  | openssl enc -aes-256-cbc -salt -pbkdf2 \
    -pass pass:"${ENCRYPTION_PASSWORD}" \
  > /backup/myapp_$(date +%Y%m%d).sql.gz.enc
```

Decrypt and restore:

```bash
openssl enc -aes-256-cbc -d -salt -pbkdf2 \
  -pass pass:"${ENCRYPTION_PASSWORD}" \
  -in /backup/myapp_20260331.sql.gz.enc \
  | gunzip \
  | mysql -u root -p myapp
```

## Summary

Compressing MySQL backup files through inline piping to `gzip` or `zstd` is the most efficient approach, achieving 80-90% size reduction for typical SQL dumps. For daily automated backups, `gzip` or `zstd` provide the best speed-to-ratio tradeoff. Always verify compressed file integrity after creation, keep both compressed and easily restorable formats, and consider adding encryption for backups stored offsite.
