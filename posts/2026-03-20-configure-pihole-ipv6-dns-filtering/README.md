# How to Configure Pi-hole for IPv6 DNS Filtering

Author: [nawazdhandala](https://www.github.com/nawazdhandala)

Tags: Pi-hole, DNS, IPv6, Ad Blocking, Filtering, AAAA, FTL

Description: Configure Pi-hole to listen on IPv6, block ads and trackers for IPv6 clients, and serve local AAAA records on your home or lab network.

## Introduction

Pi-hole is a DNS-based ad blocker that can run on any Linux host. It uses a fork of dnsmasq (FTL) to intercept DNS queries and return NXDOMAIN or NODATA for blocked domains. IPv6 support requires configuring Pi-hole to listen on your IPv6 address and routing client DNS traffic to it.

## Installation

```bash
# Install Pi-hole
curl -sSL https://install.pi-hole.net | bash

# During installation, select the IPv6 address of your interface
# as the listening address
```

## Step 1: Configure Listening on IPv6

```bash
# After installation, edit the FTL configuration
# /etc/pihole/pihole-FTL.conf

# Pi-hole's admin UI: Settings → DNS → Interface settings
# Select "Listen on all interfaces" or a specific interface
```

```ini
# /etc/dnsmasq.d/01-pihole.conf (generated by Pi-hole)
# Verify these lines are present:
listen-address=::1
listen-address=2001:db8::1  # Your server's IPv6 address
```

## Step 2: Set IPv6 Upstream DNS

```bash
# Via admin UI: Settings → DNS
# Or via command line:
pihole -a setdns 2606:4700:4700::1111,2606:4700:4700::1001

# Verify
grep "^server=" /etc/dnsmasq.d/01-pihole.conf
```

## Step 3: Configure Router/DHCPv6

```bash
# Point clients to Pi-hole's IPv6 address for DNS
# On a typical home router (example — varies by router):
# LAN → DHCPv6 DNS Server: 2001:db8::1 (Pi-hole's IPv6 address)
# Or RA: set DNS server in Router Advertisement to Pi-hole IPv6

# Test from a client
dig AAAA doubleclick.net @2001:db8::1
# Should return NXDOMAIN (blocked)
```

## Step 4: Add Local AAAA Records (Custom DNS)

```bash
# Via admin UI: Local DNS → DNS Records
# Or add a custom dnsmasq config:
cat >> /etc/dnsmasq.d/05-pihole-custom-cname.conf << 'EOF'
address=/homeserver.local/2001:db8::50
address=/nas.local/2001:db8::51
address=/printer.local/2001:db8::52
EOF

# Restart FTL
systemctl restart pihole-FTL
```

## Step 5: Verify IPv6 Blocking

```bash
# Test that ads are blocked for IPv6 clients
dig AAAA ads.google.com @2001:db8::1
# Returns: 0.0.0.0 (blocked) or NXDOMAIN

# Check Pi-hole query log
pihole -t  # tail live query log

# Check statistics
pihole -c  # chronometer
pihole status

# Web admin
curl http://[2001:db8::1]/admin
```

## Step 6: IPv6-Only Blocking Considerations

```bash
# Some ad networks serve from IPv6-only addresses
# Ensure Pi-hole upstream resolvers return AAAA for the blocklist domains
# so they can be matched and blocked

# Check if a blocked domain has AAAA record in blocklist
pihole -q doubleclick.net
```

## Whitelist/Blacklist for IPv6

```bash
# Whitelist a domain
pihole -w good-ipv6-service.com

# Blacklist additional domain
pihole -b tracking-domain.com

# Regex blacklist
pihole --regex ".*ads.*\.example\.com"
```

## Gravity Update

```bash
# Update blocklists (includes AAAA domains)
pihole -g

# Schedule automatic updates
# Pi-hole installs a cron job automatically
cat /etc/cron.d/pihole
```

## Conclusion

Pi-hole works for IPv6 clients with the same ease as IPv4 — ensure it listens on your IPv6 address and that clients receive it as their DNS server via DHCPv6 or RA. Block rates apply equally to A and AAAA queries. Use OneUptime to monitor Pi-hole's availability and alert when the admin interface is unreachable.
